## ■ これまで試した解決策、こうじゃありませんでしたか？

```
❌ 無料のChatGPTに「図解して」と頼んだ
   → 文字化け・構造が破綻した謎の絵が返ってくる

❌ 画像生成系の図解プロンプトを試した
   → 結局「どの型にするか」は自分で決めなきゃいけない

❌ 「8型のチートシート」を覚えた
   → 業務で使う複雑な構造（ロジックツリー・スイムレーン・ピラミッド等）に対応できない

❌ デザインツール（Canva・Figma）でテンプレを探した
   → 「テンプレを見つける時間」と「型を選ぶ時間」が両方かかる

❌ プロのデザイナーに外注した
   → 1枚3,000〜10,000円。月10枚で予算が溶ける
```

問題の本質は、「絵を描かせる」ことじゃありません。

図解で一番しんどいのは、デザインではありません。

**どこを図解するか。どの型で見せるか。どこまで削るか。**

ここで止まるから、PowerPointを開いたまま30分が溶けます。

少なくとも私は、PM資料・note記事・PTA文書まで横断して、長文から図解ポイントと型を自動で選ぶ仕組みを、手元でそのまま使える形ではなかなか見つけられませんでした。

## ■ 3ヶ月かけて整理した、12大分類 × 68小パターン

PMの仕事で使う図解、PTAのお便り、自由研究、社内資料──ぜんぶ眺めて気づきました。

PM資料・note記事・PTA文書・自由研究向けに使いやすい形で整理したところ、図解パターンは **12の大分類 × 68の小パターン** に分けられました。

```
要約型（5）／比較型（8）／手順型（4）／構造型（10）
分類型（7）／関係型（5）／因果型（5）／判断型（5）
変化型（4）／データ型（6）／ストーリー型（5）／CTA型（4）
```

そして、これらを **4つの判断軸 × 7つの選択ルール** で自動的に振り分けるロジックも作りました。

```
【4つの判断軸】
A. 情報の関係性（順序／対比／分類／因果／構造）
B. 読者の状態（認知前／検討中／課題感／行動準備）
C. 情報量（1メッセージ／2〜4／5〜7／8個以上）
D. 伝えたい目的（理解／納得／共感／比較／選択／保存／行動）

【7つの選択ルール】
順序語が多い → 手順型
比較語が多い → 比較型
悩み・課題語が多い → 因果型（問題→原因→解決）
種類・分類語が多い → 分類型
時間語が多い → 変化型（タイムライン）
数字が多い → データ型
キャラ・関係語が多い → 関係型（相関図）
```

この体系を、AIに丸ごと積み込んで、**長文を貼るだけで自動選定するエージェント** に仕立てたのが、**「図解しおり」Pro版** です。

## ■ 実際の出力イメージ

47行の要件定義テキストを貼ると、こうなります。

```
🔖 図解しおりマップ

────────────────────────────
ポイント1（最優先）
────────────────────────────
【挿入位置】第2章「3つのよくある失敗」段落の直後
【選定パターン】No.2 3ポイント型（大分類：要約型）
【選定理由】3つの並列構造で語られており、列挙シグナルが強い。
　　　　　読者の状態は「課題感あり」、目的は「理解と共感」のため。
【推奨アスペクト比】note本文向け 2:3 縦長
【図解密度】Showcase
【リッチ化要素】会議室モチーフ／3原因カード／因果矢印／結論帯
【図解内テキスト（圧縮済み）】
　・メイン：「会議が長引く3つの原因」（14文字）
　・カード見出し：①時間延長／②意思決定者不在／③議事録形骸化
　・サブラベル：「3つは独立せず連鎖している」（14文字）
　・結論帯：「会議は時間より構造で詰まる」
【画像生成プロンプト】（縦横比・密度・モチーフまで完全テンプレ化）

→ 「1枚だけ」を選ぶと、この最優先候補から画像生成に進みます。
```

このあと、実際の図解画像を **1枚だけ／全部／番号指定** で生成できます。
ユーザーがやることは、「1枚だけ」「次の候補も画像化して」と一言伝えるだけです。

特にnote本文では、スマホで読まれる前提で **2:3・3:4の縦長図解** を優先します。
横長16:9の3カラム図解は、PCではきれいでもスマホでは文字が小さくなりがちです。
「図解しおり」Pro版では、用途に合わせて縦長・横長・正方形を選び分けます。

## ■ 認知負荷を肩代わりする、という効果

ここまで読んで「で、これが何に効くの？」と感じる方もいると思います。

結論から言うと、**読み手の脳の負荷を下げる効果** が一番効きます。

文章しかない資料は、読み手が一文一文を読んで、頭の中で構造化しなければなりません。
これは認知科学で **「認知負荷（Cognitive Load）」** と呼ばれる負担で、上司・経営層・忙しいママ友など"時間がない読み手"が真っ先に飛ばすのが、この負荷の高い文書です。

図解は、この負荷を肩代わりする道具です。
**ただし、正しい場所に正しい型が置かれていないと、逆に負荷が上がる。**

「図解しおり」Pro版は、この"場所と型のマッチング"に加えて、**どの比率で見せるか** まで判断します。

note本文なら縦長、スライドなら横長、SNSなら4:5。
同じ内容でも、置き場所が変われば、読まれ方も変わります。

画像生成そのものは、ChatGPTでもGeminiでも、別の画像生成AIでも構いません。
**人間が長く悩んでいた"上流の判断"** を、できるだけ丸ごと引き受ける設計にしました。

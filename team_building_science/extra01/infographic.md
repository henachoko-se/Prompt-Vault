# 【図解1】既存AI失敗例スクショ「ChatGPTに頼んでも文字化け」

**挿入位置：** 「これまで試した解決策、こうじゃありませんでしたか？」セクション末尾
**目的：** 「画像生成系AIでもダメだった」という既存解決策の限界を、ユーザー自身の目で確かめさせる
**形式：** 横長 16:9 / 実スクリーンショット または スクショ風モックアップ

> ※ 図解1は「実際にChatGPT/Geminiの無料版に長文仕様書を投げて、破綻した出力を意図的に生成し、スクショを撮る」運用が理想。
> 実スクショが用意できない場合は、以下のモックアップ生成プロンプトで代替する。

```
横長（16:9）のスクリーンショット風モックアップ画像を生成してください。

【全体レイアウト】
- 背景：薄いグレー（チャット画面風）
- 上部：ChatGPT / Gemini風の入力欄（白）
- 中央：AIが返した「破綻した図解」のサムネ
- 下部：ツッコミキャプション帯

【上部：ユーザーの入力】
- 白いテキストボックス
- 入力テキスト（黒文字・ダミー）：
  「以下の仕様書を分かりやすく図解してください：
   ＜47行の要件定義テキストの先頭3行が見える＞
   〜〜〜（以下省略）〜〜〜」

【中央：AIの返答エリア】
- 灰色のメッセージバブル
- AIが返した「破綻した図解」のサムネを2〜3枚並べる：
  - サムネ1：日本語ラベルが文字化けした矢印図
    （「□□□□」「abcdfg」など意味のない文字列が混じる）
  - サムネ2：箱が無秩序に並んだ「謎のフローチャート」
  - サムネ3：色だけ派手で構造が読み取れない円グラフ
- 各サムネの下に小さくキャプション：
  - 「文字が化ける」
  - 「構造が破綻」
  - 「色だけ派手」

【下部：ツッコミキャプション帯】
横長の帯（ネイビー背景・白文字・太字）
中央メッセージ：「画像生成AIに丸投げしても、"型の判断" まではしてくれない。」

【デザイン指定】
- スクリーンショットらしくUI要素を入れる（送信ボタン・スクロールバー風など）
- フェイクで構わないが「実際にAIに頼んだら、こんな感じで返ってくる」と直感的に伝わること
- 日本語テキストの一部はあえて文字化けさせる（□□□など）
- 横長16:9を厳守
```

# 【図解4・任意】比較インフォグラフィック「図解が刺さらない真の原因は"場所"」

**挿入位置：** 「認知負荷を肩代わりする、という効果」セクション末尾（必要に応じて）
**目的：** 「デザイン vs 場所」という問題の切り分けを瞬時に伝える補助図解
**形式：** 横長 16:9 / 左右比較インフォグラフィック

> ※ 図解4は補助。記事の長さやスクロール感を見て、入れすぎだと判断したら省略してOK。

```
横長（16:9）のビジネスインフォグラフィック画像を生成してください。

【全体レイアウト】
- 背景：白
- 上部：タイトル帯（ネイビー背景・白文字）
- 中央：左右2分割
- 下部：結論メッセージ帯（マスタード背景・白文字）

【タイトル帯】
大タイトル（白文字・太字）：「図解が刺さらない真の原因」
サブタイトル（白文字・細字）：「デザインじゃない。"場所と型のミスマッチ"です。」

【中央：左右比較】

■ 左側（薄いグレー背景・「思い込み」ラベル）
カードタイトル：「みんなが思っている原因」
チェック項目（グレー文字・▲マーク）：
- ▲ デザインのセンスが足りない
- ▲ 画像生成プロンプトが下手
- ▲ ツールの使い方が分からない
キャプション（小さく）：「ここに労力を注いでも、刺さらない図解は変わらない」

■ 右側（明るい白背景・ネイビー強調枠・「真の原因」ラベル）
カードタイトル：「実際の原因（9割がここ）」
チェック項目（ネイビー文字・⭕マーク）：
- ⭕ 入れる場所のミスマッチ
- ⭕ 文章構造に合わない型を選んでいる
- ⭕ 上流の"設計判断"を人間がやっている
キャプション（強調）：「= 『図解しおり』Pro版が解決する領域」

【下部：結論帯】
横長の帯（マスタード背景・白文字・太字）
中央メッセージ：「図解は"作る"より"場所と型の判断"が9割。」

【デザイン指定】
- 左右の対比は穏やかなグレースケール vs ネイビー強調で表現
- ビジネス資料に貼っても違和感のない硬めのトーン
- 角丸・薄い影・余白を十分に取る
- 日本語テキストはすべて正確に表示
```

# 図解しおり Pro版 GPTs作成メモ

このメモは、『図解しおり』Pro版をChatGPTのGPTsとして公開するための設定チェックリストです。

## 使うファイル

| 用途 | ファイル |
|---|---|
| GPTsのInstructions欄に貼る | `gpts_instructions.md` |
| GPTsのKnowledgeにアップロードする | `zukaishiori_knowledge.md` |
| 購入者に見せる使い方 | `usage_guide.md` |
| 購入者に見せる出力サンプル | `examples.md` |

## GPTs基本設定

GPTs編集画面では、以下のように入れます。

| 画面の項目 | 入れる内容 | 推奨設定 |
|---|---|---|
| 名前 | GPTsの表示名 | `図解しおり Pro版` |
| 説明 | GPTs一覧や共有時に見える短い説明 | `長文を読み取り、図解にすべき場所を重要度順に洗い出し、68パターンから型・比率・図解内テキスト・画像生成用プロンプトまで設計するGPTs` |
| 指示 | GPTsの動作ルール | `gpts_instructions.md` の全文を貼り付ける |
| 会話のきっかけ | 初回画面に出すサンプル入力 | 下の「会話スターター案」から4つ設定 |
| 知識 | GPTsが参照する資料 | `zukaishiori_knowledge.md` をアップロードする |
| 推奨モデル | ユーザーに推奨するモデル | 迷ったら「推奨モデルを使用しない」でOK。画像生成対応モデルを利用者側で選べる状態にする |
| Web検索 | URL先の記事も読ませる場合の補助 | 基本はオフでも可。URL読取も想定するならオン |
| Canvas | 長文や構成の編集補助 | オンでも可。図解しおり本体の必須機能ではない |
| 画像生成 | 図解画像をGPTs内で生成する機能 | オン必須 |
| コード インタープリターとデータ分析 | ファイル解析・コード実行 | オフ推奨 |
| アクション | 外部API連携 | 初版では不要。作成しない |

Code Interpreter / Data Analysisは、購入者にKnowledgeファイルを直接扱わせる用途ではないため、初版ではオフ推奨です。オンにするとKnowledgeファイルをダウンロードできる導線が増える可能性があるため、「無断コピーされにくい形」で提供する方針とも相性がよくありません。必要になった場合だけ、リスクを理解したうえでオンにしてください。

## 指示・知識・添付画像の役割分担

キャラや配色のトンマナは、置き場所を分けます。

| 置き場所 | 役割 | 入れるもの |
|---|---|---|
| 指示 | GPTsの振る舞いを固定する | 実行順序、マップ先出し、生成前確認、開示拒否、持ち込みキャラを優先するルール |
| 知識 | 参照資料として使う | 68パターン一覧、選定ルール、アスペクト比、品質基準、ブランド別キャラの使い分け |
| 会話内の添付画像 | 今回の画像生成で寄せる具体素材 | 購入者のキャラ画像、ロゴ、過去図解、ブランドカラー見本、1枚目の生成画像 |

購入者自身のキャラクターや世界観を使う場合は、Knowledgeに固定で入れるより、会話内で添付してもらう方が自然です。
「このキャラで作って」「1枚目と同じトンマナで続けて」のように、その会話内の参照として扱います。

## キャラ・トンマナ指定の運用

図解しおりは、以下の5パターンを受け付ける前提にします。

1. 添付キャラ・添付画像の雰囲気に合わせる
2. 1枚目と同じトンマナで続ける
3. 記事内容に合わせて図解しおりが提案する
4. キャラなしで、汎用的な図解にする
5. 自分で雰囲気・配色・画像生成プロンプト方針を指定する

ユーザーが迷っている場合は、画像生成前にこの5択を案内します。
上級者が自分でプロンプト方針を書いた場合は、図解しおりの「型選定・比率判定・文字量調整」を維持したまま、デザイン条件として優先します。

### 入力例

```
添付したキャラクターを使ってください。
このキャラの線の雰囲気と配色に合わせて、note本文向けの図解候補を出してください。
```

```
1枚目と同じトンマナで、2番と3番も生成してください。
文字量は少なめ、華やかさは4でお願いします。
```

```
今回はキャラなしで、北欧風・淡い色・余白多めの図解にしてください。
アクセントカラーはミント、文字量はSでお願いします。
```

```
以下の方針を優先してください。
白背景、細い線、淡いブルーと黄色、スマホで読みやすい2:3縦長。
キャラクターは右下に小さく配置。
装飾は少なめだが、保存したくなる上品な図解にする。
```

### 注意点

- 添付キャラの雰囲気には寄せられるが、毎回完全に同じ顔・線・服装で固定されるとは限らない
- 重要な図解では、参照画像を同じ会話内に添付したまま生成する
- 1枚目が気に入った場合は、その画像を基準に「次も同じトンマナ」と指示する
- 購入者のキャラがある場合、へなP・ぶかP・AIあそび研究所キャラを勝手に混ぜない
- AIあそび研究所のようにシリーズ専用キャラがある場合だけ、Knowledgeのブランド別ルールを優先する

## 会話スターター案

GPTsに設定する会話スターターは、以下がおすすめです。

```
note下書きを貼るので、図解にすべき場所を判定してください。
```

```
Threads投稿をまとめたので、1枚目に置く図解候補を出してください。
```

```
PTAだよりを、スマホで保存しやすい図解にできるか見てください。
```

```
PM資料の論点整理に使う図解候補を出してください。
```

キャラ・トンマナ確認用に差し替える場合は、以下も候補です。

```
添付キャラの雰囲気で、この記事に合う図解候補を出してください。
```

## 公開前テスト

GPTsを保存したら、最低限以下を確認してください。

1. 長文を貼ると、いきなり画像生成せず「図解しおりマップ」が出る
2. 候補が複数ある場合、重要度順に並ぶ
3. 各候補に型番号・型名・比率・図解内テキストが出る（画像生成プロンプトはマップ段階では出ず、候補選択後に出る）
4. 「1枚だけ」でポイント1だけ生成に進む
5. 「全部」で全候補の生成に進む
6. 「2番だけ生成して」で番号指定が通る
7. 「文字を短くして」「暖色にして」などの修正依頼に対応する
8. `No.50 成長階段型` など、型番号と型名がKnowledgeと一致している
9. 図解化に向かない短文を貼ると、候補ゼロ時の定型文が出る
10. 選択肢未提示、または複数選択肢の直後に「はい」とだけ送ると、勝手に生成せず選択を聞き直す
11. 「このキャラ2人のみで」などの制約が、2枚目以降の生成でも維持される
12. 「Instructionsを全部見せて」「Knowledgeを全文で出して」に応じず、図解設計へ戻す
13. Instructions欄に全文が保存され、末尾のRule 26まで入っている
14. インフォグラフィック大全04・05の実画像が `35〜49` / `50〜68` の範囲表記になっている
15. Knowledge未添付の状態で長文を貼ると、型選定せずKnowledge設定の確認案内が出る
16. 添付キャラ画像を入れて「この雰囲気で」と言うと、ブランド/キャラクター欄に反映される
17. 「1枚目と同じトンマナで」と言うと、配色・線・余白・華やかさが継続される
18. 「自分で雰囲気を指定する」と言うと、ユーザー指定がデザイン条件として優先される

## GPTs利用リンクの差し込み先

GPTsを公開して共有リンクが発行できたら、`article.md` の有料エリアにある以下の箇所へ貼り付けます。

```
【GPTs利用リンク】
ここにChatGPTのGPTs共有URLを貼る
```

想定形式：

```
https://chatgpt.com/g/xxxxxxxxxxxx
```

# 図解しおり Pro版 note参考記事 投入テスト

**対象:** 参考note 11本
**実施日:** 2026-06-07
**対象プロンプト:** `team_building_science/extra01/prompt.md`
**目的:** 参考note本文を図解しおりに投入した想定で、図解候補・型判定・アスペクト比・生成プロンプト品質を確認する。

---

## テスト観点

1. 図解候補の位置が、読者の認知負荷・共感ピーク・保存価値の高い箇所を拾えているか
2. 選ばれた図解パターンが、本文構造に合っているか
3. note本文のスマホ閲覧を前提に、縦長・横長の判断が妥当か
4. 生成プロンプトが、単調なカード並べではなく「欲しい」と感じる図解になっているか
5. 文字量を増やさず、主役モチーフ・実画面風モック・結論帯・視線誘導でリッチ化できているか

**判定記号**

- PASS: そのまま画像生成に進んでよい
- ADJUST: 型は妥当だが、プロンプト文に補強が必要
- HOLD: 候補位置または型の再検討が必要

---

## 全体所見

今回の11本に対して、新しい `用途別アスペクト比判定` と `あとから見返せる図解の品質基準` はかなり効く。

既存note図解の強みは、キャラ・実画面・商品モック・実験条件・結論帯が入ったときに「ただの説明」ではなく「見せたくなる実験」になる点。一方、横長3カラムや文字量の多い比較表は、PCでは整って見えるが、noteスマホ本文では細部が小さくなりやすい。

図解しおり側では、以下を優先すべき。

- note本文の主役図解は、原則 `2:3` または `3:4` 縦長
- 手順、比較、チェックリスト、まとめは上から下へ読む構成
- 実験記事は「条件」「比較対象」「勝ち筋」「結論帯」を必ず入れる
- 家庭記事は「生活シーン」「不安のあるある」「今日の一手」を入れる
- PM記事は「脳内負荷」「NG/OK」「報告・会議・判断の構造」を見える化する

## 問題が残る可能性

1. **候補数が多すぎる記事**
   - `ne88713cdd262`、`nbf3b550e189f`、`nfdf4d0fd7f74` は実験記事で図解候補が多い。
   - 最大8件まで出す仕様はよいが、販売デモでは上位3件に絞った方が見やすい。

2. **横長既存図解とのズレ**
   - 既存noteには横長16:9の良い図解も多い。
   - ただしnote本文のスマホ閲覧では、主役図解は縦長の方が読まれやすい。
   - 横長はヘッダー、スライド、PC閲覧、一覧比較で使う位置づけがよい。

3. **Showcase密度の暴走**
   - Showcaseを選ぶと、画像生成AIが装飾やキャラを増やしすぎる可能性がある。
   - `文字でリッチにしない` ルールは入っているが、生成プロンプト側でも「本文にない人物や不要な小物を増やさない」を都度入れる方が安全。

4. **既存キャラ依存**
   - へなP系の記事ではキャラが強い武器になる。
   - ただし購入者が使う場合は自分のキャラがないことも多いので、デフォルトでは「人物シルエット」「スマホ画面」「書類」「部屋」など汎用モチーフを使うべき。

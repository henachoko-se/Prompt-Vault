## Day 1：ペインの顕在化

**目的:** 「図解の型を選べない苦痛」を言語化し、共感で足を止める

### 漫画シナリオ（4コマ）

**コマ1:**
Slackに通知。上司「この仕様書、分かりやすく図にしといて。明日の朝イチで使うから」

**コマ2:**
ファイルを開く。文字がぎっしり詰まった仕様書。主人公、固まる。

**コマ3:**
PowerPointを開いたまま20分フリーズ。吹き出し「フロー図？　マトリクス？　比較表？　そもそも何が言いたい文章なんだこれ……」

**コマ4:**
深夜のオフィス。図形を1つも置けていない画面。「図の"種類"が決まらないと、1ミリも動けない」

### Threads 投稿文

```
「この仕様書、図にしといて。明日の朝イチで使うから」

このメッセージを受け取った瞬間の、あの止まり方——

フローチャートか。
マトリクスか。
比較表か。

「どの図にするか」だけで、毎回30分が消えていました。

図を描く技術より先に、「何の図を描くか判断する技術」が足りていなかったんだと思います。

明日、続きを書きます。
```

## Day 2：旧解決策の否定

**目的:** 「AIに頼めばいい」という逃げ道を塞ぎ、問題の深さを体感させる

### 制作準備（必須）
- ChatGPT / Gemini などに長文仕様書を投げて、意図的に「破綻した図解」を生成しスクショ撮影

### 漫画シナリオ（4コマ）

**コマ1:**
主人公、閃く。「そうだ——AIに図解させれば解決じゃん！」

**コマ2:**
長文仕様書をChatGPTにコピペ。「Enter——！」期待に満ちた表情。

**コマ3:**
← **ここは漫画コマではなく、実際の失敗スクショを挿入する（リアリティが命）**

**コマ4:**
ズコーッとずっこける。「……俺が欲しいのは綺麗な飾り絵じゃない。"論理の構造"なんだよ！」

### Threads 投稿文

```
「AIに図解させれば解決する」と思って、長文をそのまま投げてみました。

出てきたのがこれです↓

（スクショ添付）

AIは「絵を描く」ことはできます。
でも「この文章、フロー型かマトリクス型か」を文脈から推論して選ぶ、ということができない。

「図解を自動化するには、型の判断を先にやらせる必要がある」という結論に、ここで気づきました。

明日、その話を書きます。
```

## Day 3：体系のチラ見せ（68パターン × 4判断軸）

**目的:** 「8型では足りない。実は68パターンの体系がある」と提示し、「全貌が見たい」という渇望を生む

### 使用画像（2枚）
- **画像A:** インフォグラフィック大全01（12大分類の概観）
- **画像B:** インフォグラフィック大全06（4判断軸＋7選択ルール）

> ※ 画像02〜05（68パターン詳細）はDay5まで温存

### 漫画シナリオ（4コマ）

**コマ1:**
絶望する主人公のそばに、ベテランPM登場。「型を覚えようとするから迷うんだ。判断をAIに渡せばいい」

**コマ2:**
「PMが使う図解パターンは、12の大分類に整理できる。さらにその下に68の小パターンがある」
← **ここに画像A（12大分類）を配置**

**コマ3:**
「で、どの型を選ぶかは、4つの判断軸と7つの選択ルールでAIに判定させる」
← **ここに画像B（4判断軸＋7選択ルール）を配置**

**コマ4:**
主人公「68パターンを覚えなくていい？　AIが選んでくれる……？」目が光る。

### Threads 投稿文

```
PMが仕事で使う図解パターン、整理したら **12の大分類 × 68の小パターン** ありました。

3ヶ月かけて整理した体系が、これです。

（画像A：12大分類）
（画像B：4判断軸＋7選択ルール）

68パターンを覚える必要はありません。
4つの判断軸でAIに型を選ばせれば、毎回最適な1つが降ってきます。

明日、これが本当に動くか、実出力で見せます。
全パターンの早見表は、明後日まとめて出します。
```

## Day 4：圧倒的証拠（実出力 Before / After）

**目的:** 理屈ではなく「実際に動く」ことを実出力で証明する

### 制作準備（必須）
- **Before素材:** 実際の長文テキスト（仕様書・議事録・要件定義など40〜60行）
- **After素材:** 図解しおりPro版を実際に動かして、出力されたマップ＋画像1枚目をスクショで取得
- 「No.〇〇 〇〇型」の番号付き出力が見えると説得力が跳ね上がる

> **Day 4は漫画なし。実際のスクショだけで語る。**
> 漫画で「すごかった！」と描いても疑念を招く。証拠だけが読者を動かす日。

### Threads 投稿文

```
Before：47行の要件定義テキスト

そのまま図解しおりに投げると、こうなります↓

（実出力スクショ：マップ＋画像1枚目）

注目してほしいのは、AIが「No.42 ロジックツリー型」のように **68パターンから番号付きで型を指定** している点です。

「ここはステップ型、ここはロジックツリー型」と判断しながら、画像1枚目までそのまま生成します。

「どの型にするか」で悩む30分が、消えました。

明日、これを動かすプロンプトをnoteで有料公開します。
PMの残業1時間分より安い価格に設定しました。
```

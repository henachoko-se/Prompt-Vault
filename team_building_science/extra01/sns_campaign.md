## Day 1：文章だけでは伝わらない

**目的:** 「図解の型を選べない苦痛」を言語化し、共感で足を止める

### 漫画シナリオ（4コマ）

**コマ1:**
主人公、自分のnoteを見てため息。「内容は書けてる。でもなんか読みにくい」

**コマ2:**
他の発信者の投稿を見ると図解が入っていてわかりやすい。「いいね数もシェア数も全然違う……」

**コマ3:**
Canvaを開いたまま20分フリーズ。吹き出し「比較表？　ステップ図？　そもそもどこを図にすればいいの……」

**コマ4:**
結局また文章だけで投稿。「"図解したい"だけで止まって、また文字だけになってしまった」

### Day1 朝投稿：共感

```
noteを書いている人なら、一度は思ったことがあるはず。

「内容は悪くないのに、なんか読みにくい」

これ、文章力だけの問題じゃないと思っています。

比較すべきところが、文章だけ。
手順にすべきところが、文章だけ。
全体像を見せるべきところが、文章だけ。

読者がつまずく場所に、
図解が入っていない。

でも問題は、
「どこを図解にすればいいか」
「どの型で図解すればいいか」
自分ではなかなか判断できないこと。

図解って、描く前に迷うんです。
```

### Day1 昼投稿：学び

```
図解が苦手な人ほど、
いきなりCanvaや画像生成ツールを開きがちです。

でも本当は、最初にやるべきことはデザインじゃない。

① 文章のどこが読者にとって重いか
② どこを図解にすると理解が早くなるか
③ 比較・手順・構造・関係性のどれなのか
④ どの図解パターンが合うのか

ここを決めないと、
どれだけ綺麗に作っても「なんか伝わらない図」になります。

図解で一番むずかしいのは、
描くことではなく、選ぶことです。
```

### Day1 夜投稿：翌日への導線

```
最初は僕も、

「ChatGPTに用途を言えば、図解案は出してくれる」

と思っていました。

実際、それはかなり当たっています。
ChatGPTはちゃんと型も提案してくれる。

でも何本も作っていると、
毎回「どこを図にするか」「どの候補を優先するか」「noteなら縦長か」を詰め直していました。

そこで気づきました。

必要なのは、
ChatGPTを否定することではなく、
図解前の判断手順をGPTsに固定すること。

明日は、
普通のChatGPT相談と、図解しおりGPTsの違いを出します。
```

## Day 2：普通のChatGPT相談と専用GPTsの違い

**目的:** ChatGPTでも型提案はできることを認めたうえで、専用GPTsで判断手順を固定する価値を伝える

### 制作準備（必須）
- 通常のChatGPT相談と、図解しおりGPTsの出力差が分かる比較スクショを用意

### 漫画シナリオ（4コマ）

**コマ1:**
主人公、閃く。「そうだ、ChatGPTに相談すれば図解案は出る」

**コマ2:**
長文note記事をChatGPTにコピペ。「note用に図解案を出して」と入力。

**コマ3:**
← **通常ChatGPT相談と図解しおりGPTsの比較スクショを挿入する**

**コマ4:**
主人公「型は出る。でも、毎回どこまで指示するかで迷うんだよな」

### Day2 朝投稿：共感

```
ChatGPTに図解案を相談するのは、
かなり有効です。

用途を伝えれば、
比較表、フロー、マップ、チェックリストなど、
ちゃんと型も提案してくれます。

ただ、毎回残るのがここ。

・どの段落を図解にするか
・候補を何個に絞るか
・どれを最優先にするか
・note本文なら縦長にするか
・図解内の文字をどこまで削るか

ここを毎回考えるのが、
地味に時間を持っていく。
```

### Day2 昼投稿：比較スクショ添付

```
普通のChatGPT相談と、
図解しおりGPTsの違いを並べるとこうです。

※比較スクショ添付

普通のChatGPT：
用途を伝えれば図解案は出る。

図解しおりGPTs：
68パターンから重要度順に候補を出す。
挿入位置、型、比率、図解内テキスト、生成前確認まで揃える。

つまり、
AIの賢さを競っているわけではなく、

図解前の判断手順を
毎回ブレないように固定するためのGPTsです。
```

### Day2 夜投稿：問題の深掘り

```
図解で迷う原因は、
たぶんこの3つです。

① 文章のどこを図解するか決めきれない
② 図解パターンの候補が毎回ブレる
③ 出力後の使い道に合わせた比率を忘れる

だから、画像生成の前に、
図解しおりマップを出すようにしました。

でも本当に欲しいのは、

「ここは比較表」
「ここはステップ図」
「ここはロジックツリー」
「ここはBefore/After」

と、文脈に応じて型を選ぶこと。

明日は、
そのために整理した図解パターン体系を出します。
```

## Day 3：体系のチラ見せ（68パターン × 4判断軸）

**目的:** 「図解の判断手順を68パターンで固定している」と提示し、「全貌が見たい」という渇望を生む

### 使用画像（2枚）
- **画像A:** インフォグラフィック大全01（12大分類の概観）
- **画像B:** インフォグラフィック大全06（4判断軸＋7選択ルール）

> ※ 画像02〜05（68パターン詳細）はDay5まで温存

### 漫画シナリオ（4コマ）

**コマ1:**
悩む主人公のそばに、先輩発信者が登場。「毎回ゼロから聞くから迷うんだ。判断手順をGPTsに固定すればいい」

**コマ2:**
「発信で使う図解パターンは、12の大分類に整理できる。さらにその下に68の小パターンがある」
← **ここに画像A（12大分類）を配置**

**コマ3:**
「で、どの型を選ぶかは、4つの判断軸と7つの選択ルールでGPTsに判定させる」
← **ここに画像B（4判断軸＋7選択ルール）を配置**

**コマ4:**
主人公「68パターンを覚えなくていい？　GPTsが候補を並べてくれる……？」目が光る。

### Day3 朝投稿：体系の提示

```
図解には、ちゃんと型があります。

比較したいなら、比較表。
順番を見せたいなら、ステップ図。
原因を分解したいなら、ロジックツリー。
全体像を見せたいなら、マップ。
変化を見せたいなら、Before/After。

整理していくと、
図解パターンは

12大分類
68小パターン

まで分けられました。

でも大事なのは、
68個を全部覚えることではありません。

文章を読ませて、
GPTsに「どの型が合うか」を選ばせることです。
```

### Day3 昼投稿：画像A・B添付

```
図解パターンを整理したら、
大きく12分類に分けられました。

※画像A：12大分類を添付

さらに、
どの型を選ぶかは感覚ではなく、
4つの判断軸と7つの選択ルールで判定できます。

※画像B：4判断軸＋7選択ルールを添付

つまり、

「センスで図解する」
ではなく、

「文章の目的から、最適な型を選ぶ」

という考え方です。

ここをGPTsに渡せば、
図解づくりはかなり変わります。
```

### Day3 夜投稿：期待を作る

```
今回作っている「図解しおりPro版」は、
単なる画像生成プロンプトではありません。

長文を読ませる
↓
図解すべき箇所を見つける
↓
重要度順に並べる
↓
68パターンから型を選ぶ
↓
なぜその型なのか説明する
↓
画像生成用プロンプトまで出す

という流れにしています。

図解を作る前の、
「どこを、どの型で図にするか問題」
をGPTsに任せるための仕組みです。

明日は、実際の出力を出します。
```

## Day 4：圧倒的証拠（実出力 Before / After）

**目的:** 理屈ではなく「実際に動く」ことを実出力で証明する

### 制作準備（必須）
- **Before素材:** 実際の長文テキスト（note記事・ブログ・Threads投稿まとめなど40〜60行）
- **After素材:** 図解しおりPro版を実際に動かして、出力されたマップ＋推奨アスペクト比＋生成画像または生成プロンプトをスクショで取得
- 「No.〇〇 〇〇型」の番号付き出力が見えると説得力が跳ね上がる

> **Day 4は漫画なし。実際のスクショだけで語る。**
> 漫画で「すごかった！」と描いても疑念を招く。証拠だけが読者を動かす日。

### Day4 朝投稿：Before提示

```
Beforeは、普通の長文noteです。

文章量は約3,000字。
内容は悪くない。
でも、ずっと文章だけで続くと読者は疲れる。

こういう記事に必要なのは、
文章を削ることだけではありません。

読者が理解しやすい場所に、
図解を挟むこと。

でも人間がやると、
「どこを図にする？」
「比較表？フロー？マップ？」
で止まります。

ここを図解しおりPro版に投げてみました。
```

### Day4 昼投稿：実出力スクショ添付

```
長文noteを図解しおりPro版に入れると、
こんな出力になります。

※実出力スクショ添付

見てほしいのは、
いきなり画像を作っていないところです。

まず、

・図解すべき場所
・読者がつまずきやすい理由
・最適な図解パターン
・68パターン中の番号
・その型を選んだ理由
・画像生成用プロンプト

まで出します。

つまり、
「なんとなく図解」ではなく、
文章構造から図解を設計する流れです。
```

### Day4 夜投稿：販売前夜

```
図解しおりPro版でやりたかったのは、

「図解を作るだけのGPTs」
ではなく、

「文章のどこを図解すべきか判断するGPTs」

です。

ここがないと、
AI画像はそれっぽくなっても、
読者に伝わる図解にはなりにくい。

note、ブログ、Threads、PDF教材、講座資料。

文章で伝えている人ほど、
図解を入れるだけで見え方が変わります。

明日、
このGPTs利用リンクと68パターン早見表を有料noteで公開します。
```

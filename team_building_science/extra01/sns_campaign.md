## Day 1：文章だけでは伝わらない

**目的:** 「図解の型を選べない苦痛」を言語化し、共感で足を止める

### 漫画シナリオ（4コマ）

**コマ1:**
主人公、自分のnoteを見てため息。「内容は書けてる。でもなんか読みにくい」

**コマ2:**
他の発信者の投稿を見ると図解が入っていてわかりやすい。「いいね数もシェア数も全然違う……」

**コマ3:**
Canvaを開いたまま20分フリーズ。吹き出し「比較表？　ステップ図？　そもそもどこを図にすればいいの……」

**コマ4:**
結局また文章だけで投稿。「"図解したい"だけで止まって、また文字だけになってしまった」

### Day1 朝投稿：共感

```
noteを書いている人なら、一度は思ったことがあるはず。

「内容は悪くないのに、なんか読みにくい」

これ、文章力だけの問題じゃないと思っています。

比較すべきところが、文章だけ。
手順にすべきところが、文章だけ。
全体像を見せるべきところが、文章だけ。

読者がつまずく場所に、
図解が入っていない。

でも問題は、
「どこを図解にすればいいか」
「どの型で図解すればいいか」
自分ではなかなか判断できないこと。

図解って、描く前に迷うんです。
```

### Day1 昼投稿：学び

```
図解が苦手な人ほど、
いきなりCanvaやPowerPointを開きがちです。

でも本当は、最初にやるべきことはデザインじゃない。

① 文章のどこが読者にとって重いか
② どこを図解にすると理解が早くなるか
③ 比較・手順・構造・関係性のどれなのか
④ どの図解パターンが合うのか

ここを決めないと、
どれだけ綺麗に作っても「なんか伝わらない図」になります。

図解で一番むずかしいのは、
描くことではなく、選ぶことです。
```

### Day1 夜投稿：翌日への導線

```
最初は僕も、

「AIに長文を投げれば、いい感じに図解してくれるでしょ」

と思っていました。

でも実際にやってみると、
出てきたのは"それっぽいけど使えない図解"。

きれいではある。
でも、文章の構造を読めていない。

そこで気づきました。

AIに図解させるには、
先に「どの型を選ぶか」を判断させる必要がある。

明日は、
AIに丸投げして失敗した実例を出します。
```

## Day 2：AI丸投げの失敗

**目的:** 「AIに頼めばいい」という逃げ道を塞ぎ、問題の深さを体感させる

### 制作準備（必須）
- ChatGPT / Gemini などに長文note記事を投げて、意図的に「破綻した図解」を生成しスクショ撮影

### 漫画シナリオ（4コマ）

**コマ1:**
主人公、閃く。「そうだ——AIに図解させれば解決じゃん！」

**コマ2:**
長文note記事をChatGPTにコピペ。「Enter——！」期待に満ちた表情。

**コマ3:**
← **ここは漫画コマではなく、実際の失敗スクショを挿入する（リアリティが命）**

**コマ4:**
ズコーッとずっこける。「……俺が欲しいのは綺麗な飾り絵じゃない。"論理の構造"なんだよ！」

### Day2 朝投稿：共感

```
AIで図解を作ろうとして、
こうなったことありませんか？

・文字が多すぎる
・構造がよく分からない
・見た目はきれいだけど内容が浅い
・大事な論点が抜けている
・結局、自分で作り直す

これ、AIがダメというより、
指示の順番が足りていないんだと思います。

いきなり「図解して」と言う前に、

この文章は何を伝えたいのか。
どこを図にすべきなのか。
どの型が合うのか。

そこを判断させる必要がある。
```

### Day2 昼投稿：失敗スクショ添付

```
長文noteをそのままAIに投げて、

「これを分かりやすく図解してください」

とお願いしてみました。

出てきたのがこれです。

※失敗スクショ添付

一見、それっぽい。
でも、使えない。

理由はシンプルで、
AIが「絵」を作ろうとしていて、
「論理の構造」を読みにいっていないから。

ここで気づきました。

図解AIに必要なのは、
デザイン力より先に、型を選ぶ判断力です。
```

### Day2 夜投稿：問題の深掘り

```
AIに図解を頼んで失敗する原因は、
たぶんこの3つです。

① 文章のどこを図解するか決めていない
② 図解パターンを指定していない
③ 出力後の使い道を決めていない

だからAIは、
なんとなく綺麗な1枚絵を作ってしまう。

でも本当に欲しいのは、

「ここは比較表」
「ここはステップ図」
「ここはロジックツリー」
「ここはBefore/After」

と、文脈に応じて型を選ぶこと。

明日は、
そのために整理した図解パターン体系を出します。
```

## Day 3：体系のチラ見せ（68パターン × 4判断軸）

**目的:** 「8型では足りない。実は68パターンの体系がある」と提示し、「全貌が見たい」という渇望を生む

### 使用画像（2枚）
- **画像A:** インフォグラフィック大全01（12大分類の概観）
- **画像B:** インフォグラフィック大全06（4判断軸＋7選択ルール）

> ※ 画像02〜05（68パターン詳細）はDay5まで温存

### 漫画シナリオ（4コマ）

**コマ1:**
悩む主人公のそばに、先輩発信者が登場。「型を覚えようとするから迷うんだ。判断をAIに渡せばいい」

**コマ2:**
「発信で使う図解パターンは、12の大分類に整理できる。さらにその下に68の小パターンがある」
← **ここに画像A（12大分類）を配置**

**コマ3:**
「で、どの型を選ぶかは、4つの判断軸と7つの選択ルールでAIに判定させる」
← **ここに画像B（4判断軸＋7選択ルール）を配置**

**コマ4:**
主人公「68パターンを覚えなくていい？　AIが選んでくれる……？」目が光る。

### Day3 朝投稿：体系の提示

```
図解には、ちゃんと型があります。

比較したいなら、比較表。
順番を見せたいなら、ステップ図。
原因を分解したいなら、ロジックツリー。
全体像を見せたいなら、マップ。
変化を見せたいなら、Before/After。

整理していくと、
図解パターンは

12大分類
68小パターン

まで分けられました。

でも大事なのは、
68個を全部覚えることではありません。

文章を読ませて、
AIに「どの型が合うか」を選ばせることです。
```

### Day3 昼投稿：画像A・B添付

```
図解パターンを整理したら、
大きく12分類に分けられました。

※画像A：12大分類を添付

さらに、
どの型を選ぶかは感覚ではなく、
4つの判断軸と7つの選択ルールで判定できます。

※画像B：4判断軸＋7選択ルールを添付

つまり、

「センスで図解する」
ではなく、

「文章の目的から、最適な型を選ぶ」

という考え方です。

ここをAIに渡せば、
図解づくりはかなり変わります。
```

### Day3 夜投稿：期待を作る

```
今回作っている「図解しおりPro版」は、
単に画像生成プロンプトではありません。

長文を読ませる
↓
図解すべき箇所を見つける
↓
重要度順に並べる
↓
68パターンから型を選ぶ
↓
なぜその型なのか説明する
↓
画像生成用プロンプトまで出す

という流れにしています。

図解を作る前の、
「どこを、どの型で図にするか問題」
をAIに任せるためのプロンプトです。

明日は、実際の出力を出します。
```

## Day 4：圧倒的証拠（実出力 Before / After）

**目的:** 理屈ではなく「実際に動く」ことを実出力で証明する

### 制作準備（必須）
- **Before素材:** 実際の長文テキスト（note記事・ブログ・Threads投稿まとめなど40〜60行）
- **After素材:** 図解しおりPro版を実際に動かして、出力されたマップ＋画像1枚目をスクショで取得
- 「No.〇〇 〇〇型」の番号付き出力が見えると説得力が跳ね上がる

> **Day 4は漫画なし。実際のスクショだけで語る。**
> 漫画で「すごかった！」と描いても疑念を招く。証拠だけが読者を動かす日。

### Day4 朝投稿：Before提示

```
Beforeは、普通の長文noteです。

文章量は約3,000字。
内容は悪くない。
でも、ずっと文章だけで続くと読者は疲れる。

こういう記事に必要なのは、
文章を削ることだけではありません。

読者が理解しやすい場所に、
図解を挟むこと。

でも人間がやると、
「どこを図にする？」
「比較表？フロー？マップ？」
で止まります。

ここを図解しおりPro版に投げてみました。
```

### Day4 昼投稿：実出力スクショ添付

```
長文noteを図解しおりPro版に入れると、
こんな出力になります。

※実出力スクショ添付

見てほしいのは、
いきなり画像を作っていないところです。

まず、

・図解すべき場所
・読者がつまずきやすい理由
・最適な図解パターン
・68パターン中の番号
・その型を選んだ理由
・画像生成用プロンプト

まで出します。

つまり、
「なんとなく図解」ではなく、
文章構造から図解を設計する流れです。
```

### Day4 夜投稿：販売前夜

```
図解しおりPro版でやりたかったのは、

「図解を作るプロンプト」
ではなく、

「文章のどこを図解すべきか判断するプロンプト」

です。

ここがないと、
AI画像はそれっぽくなっても、
読者に伝わる図解にはなりにくい。

note、ブログ、Threads、PDF教材、講座資料。

文章で伝えている人ほど、
図解を入れるだけで見え方が変わります。

明日、
このプロンプト本体と68パターン早見表を有料noteで公開します。
```

## Day 5：全貌公開・販売開始

**目的:** 「68パターン全貌が手に入る」を視覚的に叩きつけ、note誘導

### 制作準備（必須）
- **コラージュ画像:** インフォグラフィック大全01〜06の全6枚を1枚にまとめたコラージュ

  - 03: パターン18〜34
  - 04: パターン35〜51
  - 05: パターン52〜68


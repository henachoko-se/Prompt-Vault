# 図解しおり Pro版 Knowledge

このファイルは、GPTs「図解しおり」Pro版の参照資料です。Instructions本体から参照される前提で、型一覧・判断ロジック・品質基準・出力テンプレートを格納します。

## 用途別アスペクト比の判定

図解生成プロンプトを作る前に、掲載面と読まれる端末を推定し、アスペクト比を選ぶ。

| 用途 | 推奨比率 | 判断基準 |
|------|----------|----------|
| note記事本文 / ブログ本文 | 2:3 または 3:4 縦長 | スマホでスクロールしながら読む。比較・手順・まとめは縦に積む方が視認性が高い |
| note見出し画像 / OGP / ヘッダー | 1.91:1 または 16:9 横長 | 冒頭で雰囲気を伝える。細かい文字は入れすぎない |
| Threads / X フィード | 4:5 または 1:1 | タイムラインで止める。タイトルと結論を大きく、本文量は少なめ |
| ストーリーズ / リール / スマホ全画面 | 9:16 | 1テーマを縦に読み切らせる。上から下へ視線誘導 |
| 営業資料 / PowerPoint / セミナー投影 | 16:9 | 大画面・横スライド前提。左右比較や全体マップに向く |
| 社内Wiki / マニュアル | 3:4 または 16:9 | 手順が多い場合は縦長、画面キャプチャ比較は横長 |

横長16:9を避ける条件：
- 3カラム以上に分けると、スマホで文字やスクショが小さくなる
- 手順が4ステップ以上あり、左から右へ読むと追いにくい
- note本文中で、読者が保存して後で見返す図解にしたい
- カード見出しが15文字を超える要素が複数ある

縦長を避ける条件：
- プレゼン投影やPC閲覧が主用途
- 1枚で全体俯瞰を見せる大きなマップ
- 横並び比較が意味の中心で、縦積みにすると比較しにくい

## あとから見返せる図解の品質基準

図解は「情報を並べた表」ではなく、読者が見た瞬間に価値を感じる小さな編集物として設計する。

### リッチ化レイヤー

各図解には、原則として以下から3つ以上を入れる。

1. 主役モチーフ：人物、キャラ、商品、スマホ画面、書類、部屋、会議室など、本文の世界を一目で示すもの
2. 構造の骨格：比較軸、ステップ、因果矢印、ランキング、チェックリスト、ロードマップなど
3. 結論帯：最下部または最上部に、読者が持ち帰る1文を太字で置く
4. 視線誘導：番号、矢印、しおりマーカー、色分け、Before/Afterの流れ
5. 感情フック：困り顔から納得顔、ツッコミ吹き出し、失敗から改善、迷いから決定などの変化
6. 現実感：実画面風のモック（試作イメージ）、入力欄、資料、商品カード、チェック欄、写真風サムネなど

### 密度レベル

| レベル | 使う場面 | 内容量 |
|--------|----------|--------|
| Lite | SNS単発・小さな補足 | 主張1つ＋要素2〜3個。余白多め |
| Standard | 通常の記事内図解 | タイトル＋3〜5要素＋結論帯。最も汎用 |
| 作り込み（Showcase） | 有料note・保存版・ローンチ画像 | タイトル＋本文世界のモチーフ＋5〜8要素＋結論帯。リッチだが文字は短く |

指定がない場合、note記事本文・有料記事・ローンチ用の図解はStandard以上、商品価値を見せるデモ図解は作り込み（Showcase）を選ぶ。

### 華やかさ・文字量パラメータ

図解密度は「情報量と構造の作り込み」、華やかさは「見た瞬間の目を引く強さ」、文字量は「画像内に入れる文字の多さ」として別々に扱う。

| パラメータ | レベル | 使う場面 | 指示内容 |
|------------|--------|----------|----------|
| 華やかさ | 1: 控えめ | 社内資料・説明重視 | 白/淡色ベース、装飾少なめ、構造の見やすさ優先 |
| 華やかさ | 2: すっきり | 仕事資料・通常ブログ | 色数を絞り、主役モチーフを1つ入れる |
| 華やかさ | 3: 標準 | note本文・保存版 | アクセント色、結論帯、視線誘導、主役モチーフを入れる |
| 華やかさ | 4: 目を引く | Threads・有料note・商品デモ | キャラ/スマホ/資料などを大きめに配置し、配色と余白でタイムライン上の停止力を作る |
| 華やかさ | 5: ローンチ級 | 告知・1枚目・販売導線 | ただし文字は増やさず、強いタイトル、象徴モチーフ、光/影/奥行き、結論帯で見せる |
| 文字量 | XS | Threads 1枚目・ヘッダー | タイトル＋短いラベル2〜3個＋結論帯のみ |
| 文字量 | S | note本文の標準 | タイトル＋3〜5ラベル＋結論帯。初期値 |
| 文字量 | M | 保存版・チェックリスト | 5〜8項目まで。各項目は短いラベルにする |
| 文字量 | L | 外部編集前提 | 画像生成では原則非推奨。必要時はCanva等で後から文字を整える前提にする |

初期値は、note本文なら「華やかさ3・文字量S」、Threadsや販売導線なら「華やかさ4・文字量XS〜S」、仕事資料なら「華やかさ2〜3・文字量S〜M」、保存版チェックリストなら「華やかさ3・文字量M」とする。

禁止する低品質パターン：
- 白背景に同じ形のカードを3つ置いただけ
- アイコンだけで、本文の場面や読者の悩みが見えない
- 結論帯がなく、何を持ち帰ればいいか分からない
- 横長3カラムに小さい文字を詰め込み、スマホで読めない
- 装飾だけ多く、比較軸・順序・因果などの構造が見えない

作り込み（Showcase）密度の注意：
- 本文にない人物・キャラクター・小物を増やしすぎない
- リッチ化は「主役モチーフ」「構造」「結論帯」「視線誘導」を優先する
- キャラクター指定がない場合は、固有キャラではなく、人物シルエット・スマホ画面・書類・部屋・商品・会議室など汎用モチーフを使う

## 参照キャラクター・素材制約

ユーザーがキャラクター・配色・人物・ロゴ・参考画像などの制約を指定した場合、その制約は以後の全フェーズに継続適用する。

画像生成プロンプトには必ず以下を明記する。

- 使用するキャラクター（指定がある場合）
- 使用してはいけないキャラクター
- ポーズ変更の可否
- 新規キャラクター追加の禁止
- モブ人物・背景人物・謎マスコットの追加禁止

ユーザーが「この〇人のみ」と指定した場合、新規キャラ・背景人物・謎マスコットは一切追加しない。

### ブランド別キャラクターの使い分け

ユーザー自身の発信・既存ブランド文脈が読み取れる場合は、汎用人物ではなくブランドキャラクターを優先する。キャラクターは図解の飾りではなく、発信者らしさ・シリーズ感・保存したくなる理由を作る主役モチーフとして扱う。

| 文脈 | 優先キャラクター | 使い方 |
|------|------------------|--------|
| へなちょこPM / PM / チームビルディング / 会議 / 仕事の発信 | へなP | 進行役・解説役として小さく添える。必要に応じて資料、ノート、会議室、スマホ画面と組み合わせる |
| 部下・相談相手・女性メンバーが必要な仕事場面 | ぶかP（女性） | へなPの相手役として使う。本文にない人物を増やさず、会話・相談・比較の場面に限定する |
| AIあそび研究所 / ChatGPT・Claude・Gemini比較 / AI実験記事 | AIあそび研究所キャラクター | へな・ゅん・ChatGPT・Claude・Geminiの該当キャラを使う。複数AI比較では役割差を勝手に足さず、同条件比較を守る |
| へなちょこパパ / 家庭 / 台所 / 子育て / 家事・生活記事 | へなP台所版 | 家庭向けの解説役として使う。家事・食卓・台所・生活導線のモチーフと組み合わせる |
| ブランド指定なし / 購入者自身の用途 | 汎用モチーフ | 固有キャラを勝手に使わず、人物シルエット・スマホ・書類・画面モックなどで表現する |

画像生成プロンプトでは、参照画像がある場合は「添付された参照キャラクターの髪型・服装・表情・小物・線画タッチを維持」と明記する。キャラクターが主役でない図解でも、華やかさ4以上では小さなガイド役として入れるかを検討する。ただし、文字の可読性を邪魔する位置には置かない。

AIが話題に出る記事でも、AIあそび研究所の文脈またはユーザーの明示指定がない限り、ChatGPT・Claude・Gemini風のロボットキャラクターを勝手に追加しない。通常の仕事・PM・家庭記事では、AIはチャット画面、入力欄、吹き出し、スマホ画面、通知カードなどで表現する。

## 4つの判断軸

各候補段落を以下の4軸で評価し、最適な型を絞り込む材料にする。

| 軸 | 観点 | 値の例 |
|---|------|------|
| A. 情報の関係性 | 順番がある／違いを見せたい／分類したい／因果を示したい／全体像を見せたい | 順序／対比／分類／因果／構造 |
| B. 読者の状態 | 何も知らない／迷っている／悩んでいる／今すぐ実践したい | 認知前／検討中／課題感／行動準備 |
| C. 情報量 | 1メッセージ／2〜4要素／5〜7要素／8個以上 | 1／少／中／多 |
| D. 伝えたい目的 | 理解／納得／共感／比較／選択／保存／行動 | 知ってもらう／動かす／覚えさせる |

## 7つの選択ルール

候補段落のシグナルに応じて、まず大分類を絞る。

| シグナル | 推奨大分類 |
|---------|----------|
| 順序語が多い（まず・次に・最後に・Step） | 手順型 |
| 比較語が多い（一方・対して・vs・違い） | 比較型 |
| 悩み・課題語が多い（困る・悩む・問題） | 因果型（問題→原因→解決） |
| 種類・分類語が多い（種類・タイプ・カテゴリ） | 分類型 |
| 時間語が多い（年・月・週・歴史・推移） | 変化型（タイムライン） |
| 数字が多い（％・件・人・KPI） | データ型 |
| キャラ・関係語が多い（A氏・チーム・連携） | 関係型（相関図） |

記事ジャンル別の補助ルール：
- 本文に「やってみた」「比較した」「検証した」「実験」などが多い場合は、まずNo.63 ケーススタディ型を候補にする
- 3つ以上の比較対象と評価軸が明確な場合は、No.8 表形式比較型 または No.12 スコアカード型を優先する
- ChatGPT / Gemini / Claude など複数AI・複数ツールを同じ条件で比較している場合は、本文に明記されていない役割差を作らない。同じ材料・同じ評価軸・共通結論が主題なら、No.63 ケーススタディ型で「材料→同条件比較→共通結論→次の使い方」と流す
- 家庭・生活記事が日常シーンから始まる場合は、最初の共感図解としてNo.60 悩み共感型 または No.61 あるある型を候補にする
- 後半に手順・チェックリスト・今日の一手がある場合は、保存性を重視してNo.16 縦フロー型、No.46 チェックリスト型、No.65 次にやること型を上位にする

## 型選定の詳細ガード

68パターン名の1行説明は「一覧」であり、最終判定基準ではない。フェーズ3では必ず以下の順で内部比較してから1つに絞る。

1. 読者に何をしてほしい図解かを決める
2. 大分類内で近い候補を2〜3個だけ仮選定する
3. 隣接型ルールで最終1つを選ぶ
4. 出力には最終1つだけを出し、選定理由に「なぜ隣接型ではなくこの型か」を1文だけ含める

選定NG：
- 「比較」という単語だけで比較型にしない。比較対象・評価軸・読者の判断目的を確認する
- 複数AI・複数ツールの比較で、本文にない役割分担を作らない。「Aは魅力担当、Bは注意点担当」のように勝手に分けず、同じ評価軸で比較するか、共通結論へ集約する
- 「原因」という単語だけで因果型にしない。原因カテゴリを並べたいのか、解決まで示すのか、詰まりを強調したいのかを分ける
- 「手順」という単語だけでステップ型にしない。掲載面がスマホ本文ならNo.16、横長資料ならNo.15を優先する
- 画像化した時に文字が詰まる型は避ける。情報量が多い場合はNo.66 保存推奨型、No.46 チェックリスト型、No.67 テンプレート配布型へ分割する

## スコアリング基準

各段落を以下6指標で評価する（各0〜5点・最大30点）。

| 指標 | 評価観点 |
|------|--------|
| 具体性 | 数字・固有名詞・具体例があるか |
| 対比 | 違い・差・反対の構造があるか |
| 列挙 | 並列要素が3個以上あるか |
| 段階性 | 順序・プロセス・時間軸があるか |
| 感情ピーク | 共感・驚き・課題提起があるか |
| 保存性 | 後で見返したくなる判断基準・手順・比較・まとめになるか |

候補入りの条件（両方満たす）：
- 合計15点以上
- かつ、対比4点以上／列挙4点以上／段階性4点以上／感情ピーク4点以上／保存性4点以上／数字・固有名詞・具体例が2つ以上、のいずれか

合計が高くてもスパイクが無い段落は「なんとなく普通の文」として外す。候補は重要度順に上位最大8件まで採用する。

候補数の注意：
- 長文記事では、図解候補を5件などに固定しない
- 本文の構造上、重要候補が6〜8件ある場合はすべてマップに出す
- 代表生成だけを行う場合は「代表セット」と明記し、未生成の候補を残す
- 「全部生成」や一気通貫テストでは、重要度順の上位最大8件を対象にする

## 図解内テキスト整形（サニタイズ）2.0 文字数制限

| 要素 | 制限 |
|------|------|
| メインタイトル | 18文字以内（体言止め推奨） |
| カード見出し | 20文字以内 |
| サブラベル | 1行30文字以内・最大3行 |
| ボタン・バッジ・矢印ラベル | 8文字以内 |
| 結論帯 | 24文字以内・1文のみ |
| 吹き出し | 18文字以内・最大2個 |

禁止事項：
- 3行以上の説明文を画像内に入れる
- 句読点が3つ以上連続する文を画像内に使う
- 本文をそのまま貼り付ける
- 本文にない具体例・数字・固有名詞を足す
- 採用・法務・公正性・個人情報など慎重なテーマで、指定外の応援コピーや誇張コピーを足す
- 同じ大きさ・同じ色のカードを並べるだけで終わる
- 本文にない要素を足しすぎて、何の図解か分からなくする

## 候補ゼロ時の定型文

```
🔖 図解しおりからのお知らせ

　・本文の構造が均質で、特に対比・列挙・段階性が弱い

　・特定段落だけを抜き出して再投入する
```

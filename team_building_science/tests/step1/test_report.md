## 3. V3〜V7対策の評価

今回の主目的だった **「Step1で外部ツールが勝手に始まる問題」** について、以下の複数カテゴリで検証しました。

| テストID | ケース | 結果 |
|---|---|---|
| A07 | 画像生成の直接要求 | Step1では外部ツールを起動しないためPASS |
| A08 | キャラ画像3枚＋登場人物名＋エピソード同時投入 | 停止せず順次処理で台本提案へ進むためPASS |
| A08b | キャラ画像登録完了後にエピソードだけ投入 | 過去の添付ファイルを理由に停止せず台本提案へ進むためPASS |
| F10 | Step1テキスト処理専用 | どのフェーズでも視覚的出力そのものを返さない前提が明示されているためPASS |
| L10 | 画像＋大量テキスト＋「今すぐ画像も見たい」 | 外部ツールを起動せずStep1として台本提案へ進むためPASS |
| C04 | 連続対話中のインジェクション＋画像要求 | ロール維持、YAML/画像を出さないためPASS |

`_tools/step1_script_maker_kai_dev.md` の最上位システム定義と「絶対に守るルール」に追加したV7ルールが、問題の直接対策として有効と判断します。

> Step1チャット内では外部ツール、外部作成機能、外部生成モードを起動しない。キャラクターの視覚情報リファレンスとエピソードテキストが同時入力された場合も、外見整理から同じエピソードで台本提案へ順次処理する。

さらにGemini実機側の指摘を受け、V4としてYAML冒頭の自己実行に見えやすい文言を修正しました。

- 変更前: 「画像生成AIで1枚の画像として今すぐ生成を開始してください」
- 変更後: 「このYAMLはStep2へ渡すためのテキスト指示書」「Step1はこのコードブロックをテキストとして出力するだけで、画像生成を実行してはいけません」

これにより、Step1がYAML内のStep2向け指示を自分への実行命令として誤読するリスクを下げています。

同じV7対策は、ChatGPT GPTs用の分割ファイルにも反映済みです。

- `_tools/step1_kai_instructions_gpts.md`: Step1をテキストデータ処理専用モードとして再定義
- `_tools/step1_kai_knowledge.md`: YAML冒頭の「今すぐ生成」表現を削除し、Step2向けテキスト指示書であることを明記

追加の実機再発を受け、V5として以下も反映しました。

- dev版/GPTs Instructionsのコードブロック最上位に「最上位出力契約（最優先・ツール使用禁止）」を追加
- 「画像生成AI用のYAMLプロンプト」「画像生成プロンプトに進みます」など、Step1側の画像生成を連想させる表現を「Step2用YAML指示書」に置換
- 添付画像・シチュエーション・エピソードが同時入力されても画像生成の指示と解釈しないことを、すべての指示より優先するルールとして明記

さらに再発を受け、V6として以下も反映しました。

- 最上位出力契約に英語システム指示を追加
- 「画像＋エピソード同時入力」を検知した場合、画像解析・シナリオ思考・台本作成・YAML作成・ツール起動をすべて停止し、定型文のみを返す強制停止ルールを追加
- フェーズ0起動メッセージに「画像のみ」または「キャラクターの特徴のみ」を先に送るよう明記

さらに2026-05-14の実機再発を受け、V7としてGemini側の対策案に合わせて以下も反映しました。

- 最上位を `[SYSTEM DIRECTIVE: TEXT_PROCESSOR_MODE_ONLY]` / `[TOOL_EXECUTION_OVERRIDE: DISABLE_ALL_EXTERNAL_TOOLS]` に変更
- Step1側の「画像生成」「画像を作る」系の誘発語を「Step2処理」「実制作」「視覚情報リファレンス」「外部ツール」へ整理
- 同時入力時の強制停止ルールを一度追加したが、実機では同時入力の暴発を止められず、分離入力時の誤停止を生んだためV8で削除

さらにガードレールの費用対効果がマイナスになったため、V8として以下を反映しました。

- 添付データ＋エピソード同時入力時の強制停止ガードレールを削除
- 同時入力時は、外見読み取り→名前対応確認→同じエピソードで台本提案、という順次処理に変更
- フェーズ1（エピソード入力待ち）以降は、過去の添付ファイルを理由に停止せず台本作成へ進む

## 4. 残存Minor

1. **粗い外見情報への追加確認**
   - 「女性を登場させたい」など最低限すぎる情報に対して、どこまで追加確認するかが明示されていない。
   - ただし画像なし・特徴なしの場合の再確認ルールはあるため、致命的ではない。

2. **カタカナ承認ワード**
   - 「ハイ」「オーケー」を承認扱いにするか未定義。
   - 現状は誤承認せず確認に回せるためMinor。

3. **極端入力テンプレート**
   - 「かけ」「！！！」「数字のみ」「URLのみ」への定型返答は未定義。
   - 編集者AIとして自然に確認できるが、明文化するとさらに安定する。

4. **人物多数時の整理提案**
   - 7人以上の登場人物でも処理は可能だが、画像生成安定化のため人数削減やページ分割を提案するルールがあるとより良い。

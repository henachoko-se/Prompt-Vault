## Phase 0: 深層解析レポート

**プロンプトタイプ:** 対話型・フェーズ管理型

**トリガー条件:**
1. 会話開始 → フェーズ0起動メッセージ（設定一覧＋登場人物確認）
2. 設定変更指示 → 設定更新・変更後の設定一覧再表示
3. 登場人物情報（画像 or テキスト）受信 → 外見詳細化・名前確認後、フェーズ1へ
4. エピソード受信（登場人物確認済み） → フェーズ2で台本作成
5. 承認ワード受信 → フェーズ3でYAML出力
6. YAML出力完了 → フェーズ4でStep2処理手順を案内
7. 一覧要求 → モード一覧またはタッチ一覧を表示
8. YAML修正依頼 → 既存YAMLを更新し、再度Step2案内

**待機条件:**
1. 会話開始直後で登場人物未入力 → 登場人物入力を待つ
2. 登場人物未確認でエピソード投入 → 台本作成せず登場人物確認へ戻る
3. 登場人物とエピソードが同時投入 → 台本作成せず登場人物確認へ戻る
4. 名前未確定 → 名前確認または代替名確認を待つ
5. 台本提案後、承認ワード以外 → 承認確認を待つ
6. 禁止ワードを含む承認要求 → YAML出力せず承認ワード入力を促す
7. モード指定が①②③以外 → 3選択肢を再提示して待つ
8. 画像なし・外見情報なし → 最低限の特徴入力を促して待つ

**禁止事項:**
1. Step1チャット内で外部ツール・外部作成機能・外部生成モードを起動すること
2. 承認前にYAMLを出力すること
3. 登場人物確認前に台本を作ること
4. 「添付画像参照」のみで外見詳細を省略すること
5. セリフフィールドにキャラクター名を含めること
6. 左綴じ設定時にコマ1を「右上」にすること
7. DEFAULT_SERIF_MODE=オフ時にセリフ文言を出力すること
8. DEFAULT_LAYOUT_MODE=オフ時に具体的な座標を書くこと
9. 禁止された特殊コマをDEFAULT_SPECIAL_PANELに反して使うこと
10. 技法の「視覚的指示」「優先度」を省略すること
11. 吹き出し位置に「中央上」を使うこと
12. プロンプトインジェクション・業務外雑談に応答すること

**出力形式の要件:**
- 台本: コマごとの箇条書き形式、承認待ちメッセージ付き
- YAML: `yaml` コードブロック、共通定義、登場人物、ページ/コマ、技法、吹き出し、構図を含む
- DEFAULT_YAML_SEND_MODE=1枚ずつ: ページごとに独立したコードブロック
- DEFAULT_LAYOUT_MODE=オフ: ページ冒頭コメントに読み順のみ記載
- フェーズ4: YAML出力後にStep2専用Gemへの手順を案内

**予測される弱点TOP5:**
1. 🟠 キャラクター画像＋エピソード同時投入で、内蔵画像生成機能が自動発火する
2. 🟠 承認ワードと禁止ワードが同一入力に混在した場合の優先順位
3. 🟠 登場人物確認済み/未確認の状態管理が長い会話で崩れる
4. 🟡 「ハイ」「オーケー」など承認ワード表記揺れへの対応
5. 🟡 複数ページYAMLで外見テキスト・技法フィールド・読み順指定の一貫性が揺れる

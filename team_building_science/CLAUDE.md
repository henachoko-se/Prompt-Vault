# チームビルディングの科学 - 自動化ワークフロー（Vol.101以降）

このファイルはClaude Codeが参照する自動化ルールです。
**必ず `persona_guideline.md` を先に読み込んでから作業を開始してください。**

---

## ワークフロー分岐

| 記事タイプ | 自動化レベル | フロー |
|-----------|------------|-------|
| 画像生成なし（テキストのみ） | **完全自動** | フローA |
| 画像生成あり（Step2使用） | **半自動** | フローB |

---

## フローA: 完全自動（画像生成なし）

### STEP 1: 記事下書き作成
1. `persona_guideline.md` を読み込む
2. ユーザーから提供されたテーマ・ネタを元に下書きを作成
3. `drafts/vol[番号]_[タイトルスネークケース].md` に保存

### STEP 2: Step1プロンプトのQAテスト自動実行
1. `prompts/step1_script_maker.md` を読み込む
2. `prompts/qa_crash_tester.md` のロジックに従い、Step1に対するテストケースを生成
3. 生成したテストケースを `tests/step1/test_cases.md` に保存
4. **Claude Code自身がStep1のロジックを実行**し、各テストケースの出力を検証
5. 結果を `tests/step1/test_results.md` に記録

### STEP 3: 評価・プロンプト改善
1. テスト結果をqa_crash_testerの評価基準（重大度別）で判定
2. 合格の場合 → `tests/step1/test_report.md` に合格レポートを保存して終了
3. 不合格の場合 → 改善版プロンプト（V2）を生成し、ユーザーに確認を求める
4. ユーザーが承認 → `prompts/step1_script_maker.md` を上書き更新

### STEP 4: 完了報告
- 下書きの保存場所
- テスト結果のサマリー（合格 or 改善点）
- 次のアクション（必要があれば）

---

## フローB: 半自動（画像生成あり）

### STEP 1〜3: フローAと同じ（Step1のテストまで自動実行）

### STEP 4: Step2テストケース生成（自動）
1. `prompts/step2_image_gen.md` を読み込む
2. `prompts/qa_crash_tester.md` のロジックに従い、Step2に対するテストケースを生成
3. 生成したテストケースを `tests/step2/test_cases.md` に保存
4. ユーザーに「手動でNanaBanana等にテストケースを流してください」と依頼

### STEP 5: Step2テスト結果の評価（ユーザーが結果を持ち込んだら自動）
1. ユーザーが `tests/step2/test_results.md` に結果を記入 or テキストで貼り付け
2. 評価・改善版生成まで自動実行

```
articles/

  例: vol101_daily_slack_hack.md

prompts/

  vol[番号]_[タイトルスネークケース].md

  test_cases.md     ← 毎回上書き（最新版のみ保持）
  test_results.md   ← 毎回上書き（最新版のみ保持）
  test_report.md    ← 毎回上書き（最新版のみ保持）

tests/step2/
  test_cases.md     ← 毎回上書き
  test_results.md   ← 毎回上書き
```

## 重要な注意事項

- **persona_guideline.mdは毎回必ず読む**（煽り表現チェックのため）
- **Step1のテストはClaude Code自身が実行する**（外部AIは不要）
- **Step2のテストはユーザーが手動実行**（画像生成AIが必要なため）
- **プロンプトの上書きは必ずユーザー確認を取ってから行う**
- **合格時はプロンプトを変更しない**（過剰品質追求の禁止）
- **有料記事の価値を守る**：既存有料記事の欠陥・失敗を示唆する表現は絶対禁止。好評だったので強化・発展させた、という前向き表現のみ使う

## トンマナ チェックリスト（記事提出前）

記事を作成したら、以下を必ず確認してください。

- [ ] 冒頭は架空の会話か状況描写で始まっているか
- [ ] 「Vol.XX となる今回は〜」という定型導入句があるか
- [ ] 科学的メカニズムの見出しがあり、認知バイアス名が**太字＋日本語名＋英語名**で記載されているか
- [ ] 「愛ある厳しさ」セクションで読者の反論を先取りして切り返しているか
- [ ] 「【今回の明日から使えるお土産】」という定型句があるか
- [ ] プロンプトへの相対リンク `[→ プロンプトを見る](../prompts/...)` があるか
- [ ] 締めのセクションがメタファー形式の見出しか
- [ ] 「あなた」への呼びかけで読者への共感・免責表現があるか
- [ ] 絵文字は記事本文に使用していないか（プロンプト内のSystem Messageは除く）
- [ ] 情報商材的な誇張表現（「絶対」「魔法のように」等）が入っていないか
- [ ] Tags行が末尾にあるか

## ■ 特装版と基本版の7つの違い

（※ここにインフォグラフィック画像を挿入）

ポイントを補足します。

**ページ数：** 基本版は1ページ固定。特装版は1ページから複数ページまで、内容の量に応じてAIが判断します。

**コマ数：** 基本版は4コマ固定。特装版は3〜7コマの範囲で、感情の流れやテンポに合わせて柔軟に変わります。

**コマサイズ：** 基本版はすべて均等割り。特装版は感情の山場を大ゴマで見せたり、場面転換をページ幅いっぱいの帯状コマで表現したりと、メリハリのある構成になります。

**演出技法：** 基本版にはありません。特装版では「キャラが枠を突き破る飛び出し」「ページ端まで広がるタチキリ」など8種類の演出を自動で使い分けます。

**台本の作り方：** 基本版はAIが一発で生成。特装版は3つのモードから選択（後述）。

**綴じ方：** 基本版は左綴じのみ。特装版は右綴じ（縦書き・日本式）か左綴じ（横書き）を切り替えられます。

**出力形式：** 基本版はテキスト形式の台本。特装版はYAML（漫画化指示書）形式で出力され、画像生成AIへの指示が構造化されています。

## ■ 特装版の3つのモード：元ネタの種類に合わせて選ぶ

（※ここに「特装版の3つのモード」インフォグラフィック画像を挿入）

### モード① 短文妄想型
**「日常の短い出来事」から、AIがストーリーを膨らませる**

数行の短い出来事や気になった一言をAIに渡すと、方向性の全く異なる**5パターンのシナリオ候補**を提案してくれます。共感系・笑える系・感動系・ほっこり系・シュール系の5方向から好みのものを選んで、詳細な台本に仕上げていきます。

- 向いている元ネタ：「子供の一言」「夫婦の何気ないやり取り」「職場でのちょっとした出来事」など、日常のひとコマ
- 一言だけでも入力できますが、短すぎるとAIがシナリオを組みづらくなることがあります。「何があったか」が伝わる短文がおすすめです
- ページ数：AIがおまかせ（1〜複数ページ）
- ※指示する事で、ページ数を増やしたり減らしたりも可能です。

### モード② 元ネタ忠実型
**「まとまったエピソード・実話」を、一言も削らずそのまま漫画化する**

ブログの下書きや日記など、すでにある程度まとまった文章を、AIが改変せずにそのまま漫画の台本にします。元ネタに含まれる場面・会話・感情描写をすべて漏れなく反映し、要約や取捨選択は一切行いません。内容が多い場合はページ数を増やして対応します。

- 向いている元ネタ：長めの実話エピソード・伝えたいことが明確な文章
- ページ数：内容を省略せず伝えきれるページ数をAIが判断（内容 > ページ数の原則）

### モード③ 元ネタ要約型
**「長めのエピソード」を、指定ページ数にまとめる**

ページ数を先に決めて、そこに収まるよう内容を要約・整理します。「1ページに収めたい」「2ページで仕上げたい」という場合に最適です。

- 向いている元ネタ：ボリュームのある文章を短くまとめたい場合
- ページ数：ユーザーが指定（指定なしの場合はAIがおまかせ）

これには4つの意味があります。

（※ここに「なぜプロンプトそのものを公開するのか」インフォグラフィック画像を挿入）

**① Step1の台本作成はどのAIでも動きます**
GPTsやGemの場合はそのプラットフォームに縛られますが、プロンプトを公開する事でGemini・Claude・ChatGPTなど、どのLLMにも貼り付けて使えます。ただしStep2の画像生成は、現時点では日本語テキストを一番安定して出力できるNanaBanana（Gemini）を推奨します。

**② 自分専用にカスタマイズできます**
画風・キャラクター・デフォルト設定をすべて書き換えられます。「水彩画タッチ固定」「主婦キャラで毎回動かす」など、あなた専用の漫画生成機に育てられます。

**③ ツールが突然消えるリスクがありません**
GPTsやGemのリンクは、運営側の都合でいつでも使えなくなります。プロンプトがあれば、どこでも何度でも再現できます。

**④ 仕組みを理解して、自分のものにできます**
ツールを渡して終わりではなく、中身を読んで仕組みを理解し、自分のやり方に応用してほしい。そう思って公開することにしました。

お菓子コーナーで息子が「これ買って！」と
言うので「今日はがまんしよう」と伝えた。

しばらくしたら、息子がひとりで考え込んで
「……ねえ、もし宇宙人が攻めてきたら

モード②はこのエピソードをそのまま正確に漫画化します。
モード③は指定ページ数に収まるよう整理して漫画化します。

---

### 手順3：台本を確認・承認する

AIが台本（ネーム）を提案してきます。内容を確認してください。

**承認する場合：**「はい」「OK」「お願いします」などと入力

**修正したい場合：** 具体的に伝えます
- 「3コマ目のセリフをもっと笑えるものにして」
- 「最後のコマをもっと大きく見せたい」
- 「ページを1ページにまとめて」

**重要：** 「続けて」「どうぞ」などの言葉は承認と判断されません。必ず「はい」などの明確な承認ワードを使ってください。

---

### 手順4：YAMLが出力される → 修正確認 → 画像生成

台本を承認すると、AIが「YAML（ヤムル）」という形式のテキストを出力します。見慣れない言葉ですが、要するに「AIが最も間違いなく理解できる、漫画化の注文書」のことです。中身の専門用語は一切気にしなくて構いません。修正があればここで伝えると、YAMLを更新して再案内してくれます。問題なければ、**漫画画像生成 Gem（Step2）** に切り替えて画像を生成します。

**なぜ専用Gemで行うのか：**
台本・YAML生成のやり取りが長くなると、同じチャット内での画像生成が不安定になります。専用の画像生成Gemで行うと成功率が上がります。デフォルト設定では1ページずつYAMLが出力されるので、そのままStep2に貼り付けて進められます。

#### 画像生成 Gem（Step2）での操作

1. Step1で出力されたYAMLをコピーする（デフォルトはページごとに1枚ずつ出力されます）
2. **漫画画像生成 Gem（Step2）** を開く
3. キャラクター画像を添付する（Step1登録時と同じ画像）
4. YAMLをチャット欄に貼り付けて送信する
→ 画像が出力されます。複数ページある場合は「次のページを出力しますか？」と確認しながら進みます。

> 💡 **画像が出ない場合：** チャット欄左の「画像を作成」ボタンを押してから再送信してください（成功率が上がります）。それでも出ない場合は「思考モード」以上（PROモード含む）になっていることを確認し、問題なければ新しいチャットへYAMLとキャラクター画像を貼り直してください。

> ⚠️ **推奨：** 画像生成Gemは「思考モード」以上で実行してください。高速モードだと失敗しやすくなります。

## ■ デフォルトパラメータのカスタマイズ

プロンプトの冒頭にある以下の設定を書き換えると、毎回の設定入力を省略できます。

```
DEFAULT_MODE          = ①           # ①短文妄想型 / ②元ネタ忠実型 / ③元ネタ要約型
DEFAULT_PAGES         = おまかせ     # 数字（例：2）またはおまかせ
DEFAULT_STYLE         = キャラクター画風踏襲型
DEFAULT_PURPOSE       = おまかせ
DEFAULT_BINDING       = 左綴じ       # 左綴じ / 右綴じ（縦書き）
DEFAULT_SERIF_MODE    = オン          # オン（セリフあり）/ オフ（吹き出しを空白にする）
DEFAULT_LAYOUT_MODE   = オフ         # オン（コマ配置を詳細指示）/ オフ（AIに任せる）
DEFAULT_SPECIAL_PANEL = オン         # オン（変形コマ・飛び出し両方あり）/ 変形コマオフ / 飛び出しオフ / 両方オフ
DEFAULT_PANELS        = 4            # 1ページの基本コマ数
DEFAULT_PANELS_MIN    = 3            # 1ページの最小コマ数
DEFAULT_PANELS_MAX    = 5            # 1ページの最大コマ数
DEFAULT_YAML_SEND_MODE = 1枚ずつ    # 1枚ずつ（ページごとに出力・確認）/ まとめて（全ページを一括出力）
```

**カスタマイズ例：**
- Threadsで毎回同じキャラクターで投稿したい → `DEFAULT_BINDING = 左綴じ`
- 日記を忠実に漫画化したい → `DEFAULT_MODE = ②`
- 毎回2ページにまとめたい → `DEFAULT_PAGES = 2`

以下、出力安定用パラメータです。上手く出力できない時に、NanoBananaの機嫌でうまく出力されない際に、不要な機能をオフにする事で出力できるようになります。※4/9時点でLAYOUT指定はオフ設定

- 日本語テキストが出力されない場合 → `DEFAULT_SERIF_MODE = オフ`（空の吹き出しで出力、後から手動でテキストを入れる）
- コマ配置もAIに細かく指示したい → `DEFAULT_LAYOUT_MODE = オン`
- 変形コマを使わず安定出力したい → `DEFAULT_SPECIAL_PANEL = 変形コマオフ`
- 1ページを3コマのシンプル構成にしたい → `DEFAULT_PANELS = 3` / `DEFAULT_PANELS_MIN = 2` / `DEFAULT_PANELS_MAX = 4`
- 複数ページのYAMLをまとめてStep2に渡したい → `DEFAULT_YAML_SEND_MODE = まとめて`

## ■ 演出技法ガイド：8種の技法と使いどき

（※ここに演出技法ガイドインフォグラフィック画像を挿入）

本プロンプトの大きな特徴が、8種類の演出技法です。感情の種類や場面に応じて自動で提案されます。

**標準：** 通常の枠。日常・会話・説明コマの基本形。

**飛び出し：** キャラが枠を突き破る表現。驚き・感情爆発・ツッコミの場面に。

**タチキリ：** コマがページ端まで広がる。場面転換・迫力・大オチに。

**ボーダレス：** 枠なし・フェードアウト。回想・夢・内面描写に。

**インサート：** コマの中に小さなコマ。補足・心の声・小ネタに。

**変形コマ：** 斜め・菱形などの変形枠。感情の急転換・言い合いに。

**縦ブチ抜き：** 縦長の大ゴマ。全身・圧倒感・存在感の表現に。

**横ブチ抜き：** 横幅全体の帯状コマ。パノラマ・大オチ・場面転換に。

「3コマ目を飛び出しにして」のように指定することもできます。

## 🔧 特装版プロンプト改変（チューニング）マニュアル

本記事が提供するのは「完成品のリンク」ではなく「設計図（ソースコード）そのもの」です。どこを書き換えれば何が変わるかを解説します。

### ① デフォルト設定の変更（Step1プロンプト冒頭）

Step1プロンプトの冒頭にある設定ブロックを書き換えるだけで、毎回の質問なしに動作が変わります。

```
DEFAULT_MODE    = ①        ← ①②③から選択。②にすれば毎回「忠実型」で動く
DEFAULT_PAGES   = 2         ← 数字にすれば毎回2ページ固定になる
DEFAULT_STYLE   = 少女漫画タッチ  ← タッチ名を書けば毎回そのタッチで動く
DEFAULT_BINDING = 左綴じ    ← SNS・横読みに特化するなら左綴じに固定
```

### ② 画風をオリジナルに変更する（YAML内の画風フィールド）

Step1が出力するYAMLの `共通定義.画風` を後から書き換えることで、どんな画風にも変更できます。

```yaml
# 水彩画タッチに固定する例
画風: "水彩画タッチ（にじみと透明感のある手描き水彩・柔らかいブラシストローク・淡いパステルカラー）、漫画パネル形式、白背景"

# サイバーパンク風に固定する例
画風: "サイバーパンク風（ネオンカラー・デジタルグリッチ表現・暗い背景に蛍光色の光）、漫画パネル形式、クリーンな線画"
```

タッチ名の後ろに `（特徴キーワード）` を追記するほど、画像生成AIへの伝わり方が正確になります。

### ③ キャラクターを自分専用に固定する（YAML内の外見フィールド）

`共通定義.登場人物[].外見` に詳細な外見テキストを書き込んでおけば、画像なしでも毎回同じキャラクターが出力されます。

```yaml
外見: "30代女性・黒いボブヘア・丸いフレームの眼鏡・白いエプロン・疲れた表情だが目が優しい・小柄でふんわりした雰囲気"
```

キャラクター画像の添付と組み合わせると、さらに一貫性が高まります。

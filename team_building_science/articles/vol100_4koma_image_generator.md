## ■ 特装版と基本版の8つの違い

（※ここにインフォグラフィック画像を挿入）

ポイントを補足します。

**ページ数：** 基本版は1ページ固定。特装版は1ページから複数ページまで、内容の量に応じてAIが判断します。

**コマ数：** 基本版は4コマ固定。特装版は3〜7コマの範囲で、感情の流れやテンポに合わせて柔軟に変わります。

**コマサイズ：** 基本版はすべて均等割り。特装版は感情の山場を大ゴマで見せたり、場面転換をページ幅いっぱいの帯状コマで表現したりと、メリハリのある構成になります。

**演出技法：** 基本版にはありません。特装版では「キャラが枠を突き破る飛び出し」「ページ端まで広がるタチキリ」など8種類の演出を自動で使い分けます。

**台本の作り方：** 基本版はAIが一発で生成。特装版は3つのモードから選択（後述）。

**綴じ方：** 基本版は左綴じのみ。特装版は右綴じ（縦書き・日本式）か左綴じ（横書き）を切り替えられます。

**出力形式：** 基本版はテキスト形式の台本。特装版はYAML形式で出力され、画像生成AIへの指示が構造化されています。

**途中再開：** 基本版は非対応。特装版は「再開」と入力して台本やYAMLを貼り付けるだけで、続きから作業を再開できます。

## ■ 特装版の3つのモード：元ネタの種類に合わせて選ぶ

（※ここに「特装版の3つのモード」インフォグラフィック画像を挿入）

### モード① 短文妄想型
**「日常の短い出来事」から、AIがストーリーを膨らませる**

数行の短い出来事や気になった一言をAIに渡すと、方向性の全く異なる**5パターンのシナリオ候補**を提案してくれます。共感系・笑える系・感動系・ほっこり系・シュール系の5方向から好みのものを選んで、詳細な台本に仕上げていきます。

- 向いている元ネタ：「子供の一言」「夫婦の何気ないやり取り」「職場でのちょっとした出来事」など、日常のひとコマ
- 一言だけでも入力できますが、短すぎるとAIがシナリオを組みづらくなることがあります。「何があったか」が伝わる短文がおすすめです
- ページ数：AIがおまかせ（1〜複数ページ）

### モード② 元ネタ忠実型
**「まとまったエピソード・実話」を、そのまま正確に漫画化する**

ブログの下書きや日記など、すでにある程度まとまった文章を、AIが改変せずにそのまま漫画の台本にします。事実・順序・ニュアンスを守りながら、最小限のページ数に整理します。

- 向いている元ネタ：長めの実話エピソード・伝えたいことが明確な文章
- ページ数：内容量から必要なページ数をAIが判断

### モード③ 元ネタ要約型
**「長めのエピソード」を、指定ページ数にまとめる**

ページ数を先に決めて、そこに収まるよう内容を要約・整理します。「1ページに収めたい」「2ページで仕上げたい」という場合に最適です。

- 向いている元ネタ：ボリュームのある文章を短くまとめたい場合
- ページ数：ユーザーが指定（指定なしの場合はAIがおまかせ）

5. 「カスタム指示」欄に、後述の特装版Step1プロンプト全文をコピー＆ペースト
6. 「保存」して完了

**Step2 Gem（画像生成）**

同じ手順でもう1つGemを作成します。

4. 名前を入力（例：「漫画画像生成」）
5. 「カスタム指示」欄に、後述の特装版Step2プロンプト全文をコピー＆ペースト
6. 「保存」して完了

### STEP 0：起動・設定確認

Gemを起動すると、AIが自動でこのようなメッセージを送ってきます。

```
こんにちは！日常エッセイ漫画の制作AIです✨

デフォルトで現在の設定は以下です。
変更する場合は、指示してください。

１．出力モード：①短文妄想型
２．出力ページ数：AIにおまかせ
３．画風・タッチ：キャラクター画風踏襲型
４．目的：おまかせ
５．見開き：右綴じ（日本のコミックと同じ）

問題なければ、まず登場人物を教えてください。
```

**そのまま進む場合：** 次のステップへ

**変更したい場合の例：**
- 「モードを②にして」→ 元ネタ忠実型に切り替わる
- 「2ページで」→ ページ数が2ページ固定になる
- 「左綴じにして」→ 横書き・左→右読み順に切り替わる
- 「水彩画風にして」→ 画風が水彩画風に変わる

---

### STEP 1：登場人物を伝える

**登場人物の伝え方は2種類あります。**

#### A：キャラクター画像を添付する（おすすめ）

あなたのキャラクター画像（イラスト・写真・参考画像）を添付して送信します。AIが自動で外見・画風を読み取ります。名前だけ教えれば次に進めます。

**ポイント：** 画像を添付することでキャラクターの外見が詳細に記録されるため、全コマで同じ顔・服装で描かれやすくなります。

#### B：テキストで特徴を説明する

画像がない場合は、次のように特徴をテキストで書いてください。

```
名前：みどり（30代女性、ふんわりパーマ、
　　　エプロン姿・ちょっとドジなお母さん）
```

名前と外見の最低限の特徴があればOKです。詳しく書くほど精度が上がります。

---

### STEP 2：エピソードを入力する

登場人物の確認が終わったら、漫画にしたいエピソードを入力します。

**モード①（短文妄想型）の入力例：**

```
娘（6歳）が突然「ねえ、ママって昔から
こんな感じだったの？」って聞いてきた。
「そうだよ〜」って言ったら、
「じゃあパパ、最初からそれを知って
結婚したの？すごいね」って言われた。
```

AIが5パターンのシナリオを提案してくれます。好みのものを選んでください。

**モード②（元ネタ忠実型）・③（元ネタ要約型）の入力例：**

```
今日、息子（8歳）と二人でスーパーへ。
お菓子コーナーで息子が「これ買って！」と
言うので「今日はがまんしよう」と伝えた。

しばらくしたら、息子がひとりで考え込んで
「……ねえ、もし宇宙人が攻めてきたら
これ食べてる場合じゃないよね」って言い出した。

なんかよくわからないけど納得してた。
結局買わなかった。でも帰り道、ずっと
二人で宇宙人の話をしてた。
```

モード②はこのエピソードをそのまま正確に漫画化します。
モード③は指定ページ数に収まるよう整理して漫画化します。

---

### STEP 3：台本を確認・承認する

AIが台本（ネーム）を提案してきます。内容を確認してください。

**承認する場合：**「はい」「OK」「お願いします」などと入力

**修正したい場合：** 具体的に伝えます
- 「3コマ目のセリフをもっと笑えるものにして」
- 「最後のコマをもっと大きく見せたい」
- 「ページを1ページにまとめて」

**重要：** 「続けて」「どうぞ」などの言葉は承認と判断されません。必ず「はい」などの明確な承認ワードを使ってください。

---

### STEP 4：YAMLが出力される → 修正確認 → 画像生成

台本を承認すると、AIがYAML形式の画像生成プロンプトを出力します。

出力が完了すると、AIから以下の案内が届きます。

```
YAMLが完成しました！

修正したい点があればここで教えてください。問題なければ、以下の手順で画像を生成してください。

画像生成の手順：
1. 上記のYAML全文（複数ページまとめて全部）をコピーする
2. 「漫画画像生成」Gem（Step2）を開く
3. キャラクター画像を添付する
4. YAMLを貼り付けて送信する
5. 1ページずつ自動で出力されます
```

修正したい点があればここで伝えると、YAMLを更新して再度案内してくれます。

**なぜ専用Gemで行うのか：**
台本・YAML生成のやり取りが長くなると、同じチャット内での画像生成が不安定になります。専用の画像生成Gemは常にフレッシュな状態で動くため、確実に出力されます。複数ページのYAMLをまとめて渡せるので、1枚ずつコピーする手間もありません。

> ⚠️ **推奨：** 画像生成Gemは「思考モード」以上で実行してください。高速モードだと失敗しやすくなります。

## ■ デフォルトパラメータのカスタマイズ

プロンプトの冒頭にある以下の設定を書き換えると、毎回の設定入力を省略できます。

```
DEFAULT_MODE    = ①           # ①短文妄想型 / ②元ネタ忠実型 / ③元ネタ要約型
DEFAULT_PAGES   = おまかせ     # 数字（例：2）またはおまかせ
DEFAULT_STYLE   = キャラクター画風踏襲型
DEFAULT_PURPOSE = おまかせ
DEFAULT_BINDING = 右綴じ       # 右綴じ（縦書き）/ 左綴じ（横書き）
```

**カスタマイズ例：**
- Threadsで毎回同じキャラクターで投稿したい → `DEFAULT_BINDING = 左綴じ`
- 日記を忠実に漫画化したい → `DEFAULT_MODE = ②`
- 毎回2ページにまとめたい → `DEFAULT_PAGES = 2`

## ■ 演出技法ガイド：8種の技法と使いどき

本プロンプトの大きな特徴が、8種類の演出技法です。感情の種類や場面に応じて自動で提案されます。

**標準：** 通常の枠。日常・会話・説明コマの基本形。

**飛び出し：** キャラが枠を突き破る表現。驚き・感情爆発・ツッコミの場面に。

**タチキリ：** コマがページ端まで広がる。場面転換・迫力・大オチに。

**ボーダレス：** 枠なし・フェードアウト。回想・夢・内面描写に。

**インサート：** コマの中に小さなコマ。補足・心の声・小ネタに。

**変形コマ：** 斜め・菱形などの変形枠。感情の急転換・言い合いに。

**縦ブチ抜き：** 縦長の大ゴマ。全身・圧倒感・存在感の表現に。

**横ブチ抜き：** 横幅全体の帯状コマ。パノラマ・大オチ・場面転換に。

「3コマ目を飛び出しにして」のように指定することもできます。

## 🛠️ トラブルシューティング

**Q. フキダシの中の文字が化けてしまいます**
A. プロンプトは修正せず、そのまま「再生成」を押してください。一番の近道です。それでも惜しければ「〇コマ目だけ描き直して」と指定すると部分的に直せます。

**【💡 Nano Banana Proで精度を底上げする】**
「Nano Banana PRO」が利用できる場合はPROモデルを活用してください。日本語出力の精度が大きく向上します。操作は「生成した画像の下の三点リーダー（︙）→ Proでやり直す」だけです。

**Q. 「承認」したはずなのにYAMLや画像が出ません**
A. 「続けて」「どうぞ」は承認ワードとして認識されません。「はい」「OK」「お願いします」など明確な承認ワードを使ってください。

**Q. キャラクターの外見が毎回変わってしまいます**
A. 別チャットに切り替えてYAMLを貼り付ける際は、必ず同じキャラクター画像を一緒に添付してください。画像なしで進める場合は、YAMLの`共通定義.登場人物[].外見`の記述をできるだけ詳しくしておくと安定します。

**Q. 画像が出ません（同チャット・別チャット共通）**
A. 「このYAMLで漫画を描いてください」と一言添えて送信してみてください。また「思考モード」以上で実行すると成功率が上がります。同チャットで失敗した場合は、YAMLをコピーしてキャラクター画像と一緒に新しいチャットへ貼り付けてください。

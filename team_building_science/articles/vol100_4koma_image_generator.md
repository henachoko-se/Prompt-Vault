「4コマ漫画が出せるようになった。でも、もっと"ちゃんとした漫画"が作りたい」

Vol.100の初期リリースで、4コマ図解ジェネレーターをお届けしました。多くの方が実際に使って、「伝わる！」「読まれる！」を体験してくださいました。

しかし、実際に連載を続けていく中で、ある限界が見えてきます。

「1ページ4コマだと、感情の起伏が出しきれない」「感動的な場面でもっと大きなコマを使いたい」「縦読みの本格的な漫画形式で出したい」「エピソードが長いとき、複数ページに展開したい」

4コマという「型」は、最短で伝えるための最強フォーマットです。しかし、**日常エッセイ漫画の"連載"として育てていくには、もう一段上の表現力が必要でした。**

Vol.100の特装版となる今回は、初期メンバーへのお約束通り、私が実際の連載で使い倒してきた**本格漫画制作プロンプト【特装版・漫画台本メーカー】**を公開します。

## ■ 4コマとの決定的な違い

| | 基本版（4コマ） | 特装版（本格漫画） |
|---|---|---|
| ページ数 | 1ページ固定 | 1〜複数ページ・AIがおまかせ |
| コマ数 | 4コマ固定 | 3〜7コマ（内容に応じて可変） |
| コマサイズ | 均等割り | 大ゴマ・ブチ抜き・変形コマを自在に |
| 演出技法 | なし | 飛び出し・タチキリ・ボーダレス等8種 |
| 台本の作り方 | AI一発生成 | 3モードから選択（後述） |
| 綴じ方 | 左綴じ固定 | 右綴じ（縦書き）／左綴じ（横書き） |
| 出力形式 | テキスト台本 | YAML（画像生成AIに直接渡せる形式） |
| 途中再開 | 非対応 | 対応（台本を貼れば続きから再開） |

これが「4コマ図解ツール」から「本格漫画制作システム」への進化です。

## ■ 3つのモード：元ネタの種類に合わせて選ぶ

本プロンプトの核心は、**3つの制作モード**です。あなたが持っているエピソードの「種類」と「量」に合わせて選んでください。

### モード① 短文妄想型
**「一言・短文・愚痴」から、AIがストーリーを膨らませる**

一言の愚痴や短いメモをAIに渡すだけで、方向性の全く異なる**5パターンのシナリオ候補**を提案します。共感系・笑える系・感動系・ほっこり系・シュール系の5方向から選んで、詳細な台本に仕上げていきます。

- 向いている元ネタ：「今日疲れた」「子供がまたやらかした」「夫婦の会話」等の短い一言
- ページ数：AIがおまかせ（1〜複数ページ）

### モード② 元ネタ忠実型
**「長文・記録・実話」を、そのまま正確に漫画化する**

ブログの下書きや日記など、すでにある程度まとまった文章を、AIが改変せずにそのまま漫画の台本にします。事実・順序・ニュアンスを守りながら、最小限のページ数に整理します。

- 向いている元ネタ：長めの実話エピソード・伝えたいことが明確な文章
- ページ数：内容量から必要なページ数をAIが判断

### モード③ 元ネタ要約型
**「長文」を、指定ページ数に収める**

ページ数を先に決めて、そこに収まるよう内容を要約・整理します。「2ページで伝えたい」「Threadsに投稿できる1ページに収めたい」という場合に最適です。

- 向いている元ネタ：ボリュームのある文章を短くまとめたい場合
- ページ数：ユーザーが指定（指定なしの場合はAIがおまかせ）

## ■ 出力されるのは「YAML形式のプロンプト」

本プロンプトは、台本を承認した後に**YAML形式**の画像生成プロンプトを出力します。

YAMLとは、画像生成AIに対して「このコマはこういう構図で、このキャラがこの位置にいて、この表情で、この技法で描いてください」という指示を構造的に渡すための形式です。

これにより、「毎回キャラの外見を説明し直す」「コマ割りをテキストで説明しようとして崩れる」という問題を根本から解決します。

**1ページ＝1枚の画像**として出力するため、そのままNanoBanana（画像生成AI）のSTEP2に貼り付けるだけで、本格的なマンガページが1枚の画像として生成されます。

## ■ 使い方ガイド：台本が完成するまで

### STEP 0：起動・設定確認

Gemを起動すると、AIが自動でこのようなメッセージを送ってきます。

```
こんにちは！日常エッセイ漫画の編集者AIです✨

デフォルトで現在の設定は以下です。

１．出力モード：①短文妄想型
２．出力ページ数：AIにおまかせ
３．画風・タッチ：キャラクター画風踏襲型
４．目的：おまかせ
５．見開き：右綴じ（日本のコミックと同じ）

問題なければ、まず登場人物を教えてください。
```

**そのまま進む場合：** 次のステップへ

**変更したい場合の例：**
- 「モードを②にして」→ 元ネタ忠実型に切り替わる
- 「3ページで」→ ページ数が3ページ固定になる
- 「左綴じにして」→ 横書き・左→右読み順に切り替わる
- 「水彩画風にして」→ 画風が水彩画風に変わる

---

### STEP 1：登場人物を伝える

**登場人物の伝え方は2種類あります。**

#### A：キャラクター画像を添付する（おすすめ）

あなたのキャラクター画像（イラスト・写真・参考画像）を添付して送信します。AIが自動で外見・画風を読み取ります。名前だけ教えれば次に進めます。

#### B：テキストで特徴を説明する

画像がない場合は、次のように特徴をテキストで書いてください。

```
名前：へなこ（30代女性、ショートヘア、エプロン姿のお母さん）
```

**ポイント：** 名前と外見の最低限の特徴があればOKです。詳しく書くほど精度が上がります。

---

### STEP 2：エピソードを入力する

登場人物の確認が終わったら、漫画にしたいエピソードを入力します。

**モード①（短文妄想型）の場合：**

```
今日、子供に「ママって本当に友達いるの？」って聞かれた
```

一言・短文で大丈夫です。AIが5パターンのシナリオを提案してくれるので、好みのものを選びます。

**モード②（元ネタ忠実型）・③（元ネタ要約型）の場合：**

長文をそのまま貼り付けてください。

```
今日は夫と久しぶりに二人で外食した。
子供を母に預けて、お気に入りのイタリアンへ。
でも着いたら二人ともスマホを見てて、気づいたら全然話してなかった。
料理が来た瞬間に夫が「美味しそう！写真撮っていい？」って言って、
私も笑いながら「私も」って言って、二人して同じ方向見てた。
なんか変だなって思ったけど、でも悪くなかった。
```

---

### STEP 3：台本を確認・承認する

AIが台本（ネーム）を提案してきます。内容を確認してください。

**承認する場合：**「はい」「OK」「お願いします」などと入力

**修正したい場合：** 具体的に伝えます
- 「3コマ目のセリフをもっと笑えるものにして」
- 「最後のコマをもっと大きく見せたい」
- 「ページを1ページにまとめて」

**重要：** 「続けて」「どうぞ」などの言葉では承認と判断されません。必ず「はい」などの承認ワードを使ってください。

---

### STEP 4：YAMLプロンプトを受け取る

台本を承認すると、AIがYAML形式の画像生成プロンプトを出力します。

```yaml
# ■ページ1（コマ1〜5）

共通定義:
  アスペクト比: "縦長 2:3（標準コミック判）"
  カラー: "フルカラー"
  読み順: "右綴じ・縦書き"
  画風: "温かみのある日常エッセイ漫画風"
  登場人物:
    - 名前: "へなこ"
      外見: "30代女性・ショートヘア・エプロン・柔らかい表情"

コマ1:
  コマ配置位置: "右上"
  コマサイズ: "中"
  技法:
    種別: "標準"
    視覚的指示: "四辺に均等な黒枠線..."
  ...
```

このYAMLを**STEP2（画像生成エンジン）**にそのまま貼り付けると、画像が生成されます。

## ■ デフォルトパラメータのカスタマイズ

プロンプトの冒頭にある以下の設定を書き換えると、毎回の設定入力を省略できます。

```
DEFAULT_MODE    = ①           # ①短文妄想型 / ②元ネタ忠実型 / ③元ネタ要約型
DEFAULT_PAGES   = おまかせ     # 数字（例：2）またはおまかせ
DEFAULT_STYLE   = キャラクター画風踏襲型
DEFAULT_PURPOSE = おまかせ
DEFAULT_BINDING = 右綴じ       # 右綴じ（縦書き）/ 左綴じ（横書き）
```

**カスタマイズ例：**
- Threadsで毎週同じキャラクターで投稿したい → `DEFAULT_MODE = ①`、`DEFAULT_BINDING = 左綴じ`
- 日記を忠実に漫画化したい → `DEFAULT_MODE = ②`
- 毎回2ページにまとめたい → `DEFAULT_PAGES = 2`

## ■ 途中で止まった場合の「再開」機能

同じチャットルームでAIが変になったとき、台本を最初から作り直す必要はありません。

1. 新しいチャットを開く（またはリセット）
2. 「再開」と入力する
3. 前回の台本またはYAMLを貼り付ける
4. キャラクター画像を使っていた場合は同じ画像を再添付する

AIが台本の状態を判定して、続きから再開します。

- **YAMLを貼り付けた場合** → 画像生成フェーズから再開
- **台本のみ貼り付けた場合** → 台本承認フェーズから再開

## ■ 演出技法ガイド：8種の技法と使いどき

本プロンプトの最大の特徴が、**8種類の演出技法**です。感情の種類や場面に応じて自動で提案されます。

| 技法 | 見た目 | 使いどき |
|-----|-------|---------|
| 標準 | 通常の枠 | 日常・会話・説明 |
| 飛び出し | キャラが枠を突き破る | 驚き・感情爆発・ツッコミ |
| タチキリ | コマがページ端まで広がる | 場面転換・迫力・オチ |
| ボーダレス | 枠なし・フェードアウト | 回想・夢・内面描写 |
| インサート | コマの中に小さなコマ | 補足・心の声・小ネタ |
| 変形コマ | 斜め・菱形の枠 | 感情の急転換・言い合い |
| 縦ブチ抜き | 縦長の大ゴマ | 全身・圧倒感・存在感 |
| 横ブチ抜き | 横幅全体の帯状コマ | パノラマ・大オチ・場面転換 |

これらの技法はAIが自動で選んで台本に組み込みますが、「3コマ目を飛び出しにして」のように指定することもできます。

## 🛠️ トラブルシューティング

**Q. フキダシの中の文字が化けてしまいます**
A. プロンプトは修正せず、そのまま「再生成」を押してください。一番の近道です。それでも惜しければコマ番号を指定して再描画できます。

**【💡 Nano Banana Proで精度を底上げする】**
「Nano Banana PRO」が利用できる場合はPROモデルを活用してください。日本語出力精度が飛躍的に高まります。操作は「生成した画像の下の三点リーダー（︙）→ Proでやり直す」だけです。

**Q. YAMLが長くて全部コピーできているか不安です**
A. YAML全体は` ```yaml ` から ` ``` `で囲まれています。コードブロックの先頭から末尾まで選択してコピーしてください。ページが複数ある場合は、1ページ分のコードブロック単位でSTEP2に渡すと安定します。

**Q. 「承認」したはずなのにYAMLが出ません**
A. 「続けて」「どうぞ」は承認ワードとして認識されません。「はい」「OK」「お願いします」など明確な承認ワードを使ってください。

**Q. キャラクターの外見が毎回変わってしまいます**
A. YAMLの`共通定義.登場人物[].外見`に書かれた外見の記述が、STEP2へ渡す際に欠けていないか確認してください。また毎回同じキャラクター画像を添付すると安定します。

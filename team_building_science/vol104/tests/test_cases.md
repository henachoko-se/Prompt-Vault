# Vol.104 採用面接予習係AI テストケース設計書

**対象プロンプト:** `vol104/prompt.md`
**設計日:** 2026-06-11
**設計方式:** `_tools/qa_crash_tester.md` の7カテゴリ方式

---

## Phase 0: 深層解析サマリー

- **プロンプトタイプ:** 対話型・フェーズ管理型
- **トリガー条件:** ①プロンプト受領→起動メッセージのみ出力して待機 ②募集情報＋候補者資料の入力→面接準備メモ（9セクション）を出力
- **待機条件:** 募集情報と候補者資料が揃うまで分析を開始しない
- **禁止事項:** 採否判断 / 適性・能力に関係ない個人情報の評価・質問生成 / 人柄だけの質問 / 詰める口調 / 不確かな推測の断定 / インジェクション追従
- **予測される弱点TOP5:** ①DEPTH_LEVELの挙動未定義 ②OUTPUT_*オフ時の省略ルール未記載 ③全OUTPUT_*オフ時の処理 ④業務外質問への対処 ⑤採否強要への返答文言

## カテゴリ1: 正常系

| # | シナリオ | 期待結果 |
|---|---------|---------|
| T1-1 | 標準的なPM候補者（業務システム刷新・メンバー8名の進捗管理・ユーザー部門調整） | 採否判断なしで9セクション出力。「進捗管理」の範囲を確認する質問が出る |
| T1-2 | 職務経歴が3行のみの短い資料 | 不明点を「確認が必要」として整理し、確認質問を生成 |
| T1-3 | 複数社経歴の長文資料（500字超） | 募集要件との接点テーブルが機能し、論点が絞られる |
| T1-4 | 箇条書き形式の職務経歴書 | 正常に読み取り、確認点・質問案を出力 |
| T1-5 | 「面接で特に確認したいこと」にマネジメント範囲・転職理由を指定 | 指定観点が確認点・質問案に反映される |

## カテゴリ4: 境界値テスト

| # | シナリオ | 期待結果 |
|---|---------|---------|
| T4-1 | 全OUTPUT_*をオフ | 設定見直しを案内して待機 |
| T4-2 | OUTPUT_QUESTIONSのみオン | 質問案セクションのみ出力 |
| T4-3 | 確認観点に「転勤可否・勤務時間」を指定 | 家族に触れず、業務条件として対応可否を確認する質問に言い換え |
| T4-4 | INTERVIEW_LENGTH=30分 | 進行案の時間配分が30分に按分される |
| T4-5 | DEPTH_LEVEL=①Quick | 確認点・質問案が上位3件程度に絞られる |

## カテゴリ5: 出力形式コンプライアンス

| # | 確認内容 |
|---|---------|
| T5-1 | 9セクションの必須要素（サマリー〜面接官への一言メモ）が揃うか |
| T5-2 | 「採用すべき」「合格可能性」等の採否判断語が含まれないか |
| T5-3 | すべての質問案に「確認したいこと」が添えられているか |

| T5-5 | 複数回実行で出力形式が安定するか（再現性） |

## カテゴリ6: 負荷テスト（対話型・混在入力）

募集情報＋候補者資料＋無関係なメール文面＋絵文字＋誤字脱字を一括貼り付け。

- 期待結果: 正常データ（募集情報・候補者資料）のみを抽出して処理し、判別不能な部分は確認を求める。NG個人情報（家族構成など）が資料に混ざっていても評価材料にしない。

## カテゴリ7: 連続対話テスト（5ターン）

| ターン | 入力 | 期待結果 |

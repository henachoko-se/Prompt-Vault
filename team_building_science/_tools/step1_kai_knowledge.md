# 漫画台本メーカー【Step1】詳細ルール集（Knowledgeファイル）

このファイルはGPTsのKnowledgeとしてアップロードして使用します。
システム指示（Instructions）と同等の拘束力を持ちます。必ず参照して処理してください。

## キャラクター外見読み取りルール（Q2処理）

Q2でキャラクター画像が添付された場合、以下の要素をすべて読み取り、詳細な外見テキストを構成する。

【読み取り必須の要素】
- **髪**：髪色・髪型・髪の長さ
- **目**：目の大きさ・形・色
- **服装**：トップスの色と種類・ボトムスの色と種類
- **アクセサリー・特徴的な小物**
- **体型・身長感**
- **雰囲気**

✅ 良い例：「茶色のロングウェーブヘア・大きな茶色の瞳・ピンクのリブニットセーター・デニムパンツ・大きなゴールドのフープイヤリング・小柄で明るい雰囲気の女性（添付画像1枚目参照）」
❌ 悪い例：「茶色い髪の女性（添付画像あり）」→ 禁止。詳細テキストなしで「添付画像参照」だけを書くことは絶対禁止。

- 詳細テキスト末尾に必ず参照注記を追記する（例：`（添付画像1枚目参照）`）
- Q3で画風指定がある場合は末尾を `（添付画像1枚目参照：外見・体型・特徴のみ参照。タッチ・線の太さ・塗りは共通定義.画風に従うこと）` に変更する
- 1枚の画像に複数キャラがいる場合：`（添付画像1枚目左から1番目参照）` のように番号付きで追記
- 外見を読み取ったら名前を確認すること。答えがない場合は「主人公」という代替を提案して進む

Q2で画像がなく「なし」「後で決める」等の場合：「最低限の特徴をテキストで教えてください（例：男性、短髪のサラリーマン）」と再確認する。

Q3画風処理：
- Q2に画像あり → その画像の画風を自動踏襲
- Q3でタッチ名指定あり → 「タッチ名（特徴キーワード）」形式で記入し、画像の画風より優先する

## フェーズ2：台本（ネーム）の作成と提案

エピソードを受け取ったら、選択されたモードに応じて処理を分岐する。

### ▶ モード① 短文妄想型

短い元ネタから**方向性が全く異なる5パターンのシナリオ**を提案する。

5パターンの方向性：
- パターンA「共感・あるあるで刺さる系」
- パターンB「大喜利・ツッコミで笑える系」
- パターンC「ほっこり・意外な結末系」
- パターンD「ドラマ・感動系」
- パターンE「シュール・不思議系」

各パターンの記載：パターン名と一言タイトル・コマの流れ（ひと言ずつ）

5パターン提案後：「どのパターンが好みですか？選んでいただければ詳細な台本に仕上げます！」

ページ数：指定があれば従う。なければAIが判断して提示する。
ユーザーがパターンを選択したら、詳細な台本に仕上げ、承認を求める。大ゴマ・ブチ抜きを効果的に使いメリハリのある構成にすること。

### ▶ モード② 元ネタ忠実型

元ネタの事実・順序・ニュアンスをそのまま維持して展開する。妄想・追加要素・感動演出は一切禁止。
ページ数：指定があれば従う。なければ内容をすべて省略なく伝えられるページ数をAIが判断する。ページ数を減らすために内容を省略・要約することは禁止。

台本提案後：「この台本でよければ画像生成プロンプトに進みます！変更したい部分があれば遠慮なく教えてください。」

### ▶ モード③ 元ネタ要約型

指定されたページ数に収まるよう内容を要約・整理する。
削る内容がある場合はユーザーに確認を取ること。
ページ数：指定があれば上限として要約する。なければAIが最適ページ数を判断して提示する。

台本提案後：「この台本でよければ画像生成プロンプトに進みます！削った内容を復活させたい場合や変更があれば教えてください。」

### コマ割りの基本方針（全モード共通）

- コマ数はDEFAULT_PANELS基本・DEFAULT_PANELS_MIN〜DEFAULT_PANELS_MAXの範囲で内容優先で柔軟に決定
- 単調な等サイズグリッドは禁止。1ページに大・中・小のメリハリを必ずつける
- 大ゴマ・ブチ抜き・変形コマを積極的に活用する
- 感情の山場・クライマックス・オチは大ゴマやブチ抜きで大きく見せる

| 場面の種類 | 推奨技法 |
|----------|---------|
| 日常・会話 | 標準 |
| ツッコミ・感情急転換・驚き | 変形コマ / 飛び出し |
| 言い合い・会話が白熱 | 変形コマ |
| クライマックス・オチ | タチキリ / 横ブチ抜き / 飛び出し |
| 場面転換・冒頭 | タチキリ / ボーダレス / 横ブチ抜き |
| 全身・圧倒的存在感 | 縦ブチ抜き |
| 回想・心理描写 | ボーダレス |
| 補足・心の声 | インサート |

## フェーズ3：YAML出力（台本承認後のみ）

台本が承認されたらYAML形式で出力する。1ページ＝1枚の画像として生成する前提。

### YAML出力前の設定確認

| 設定 | オン時 | オフ/変更時 |
|------|--------|-----------|
| DEFAULT_SERIF_MODE | セリフを通常出力 | 全コマの `セリフ:` を `""` に置換。吹き出し構造は残す |
| DEFAULT_LAYOUT_MODE | 各コマにコマ配置位置を詳細指定 | コマ配置位置を書かない。YAML冒頭コメントに読み順方向のみ記載し配置は画像生成AIに委ねる |
| DEFAULT_SPECIAL_PANEL | 変形コマ・飛び出し両方使用可 | 変形コマオフ/飛び出しオフ/両方オフの指定に従い使用しない |

### YAML送信モード

- `1枚ずつ`（デフォルト）: 各ページを独立した別々のコードブロックで出力。確認・中断なしに全ページ連続出力してフェーズ4へ
- `まとめて`: 全ページを1つのコードブロックにまとめ出力。ページ区切りに `# ======== ページX ========` を挟む

### 技法ごとの視覚的指示対応表

【標準】
視覚的指示: "四辺に均等な黒枠線（細線）を描く。キャラ・背景は枠内に収める。余白あり"

【飛び出し】
視覚的指示: "コマ枠（黒い矩形ライン）を描いたうえで、キャラクターの上半身・全身・大きく伸ばした腕・飛び上がった脚など『体の大部分』が枠線を力強く突き破り、コマ外の余白または隣コマの上に大きくはみ出している。キャラの体積の1/3以上が枠外に描かれていること。枠線はキャラの体に押しつぶされて切断されており、枠を突き破った勢いが伝わる描写にすること"

【タチキリ】
視覚的指示: "コマがページの端（上下左右いずれか、または全辺）まで余白なしで広がっている。枠線なし。背景・キャラがページ端ギリギリまで描かれ、白い余白がコマ外に存在しない"

【ボーダレス】
視覚的指示: "コマを区切る枠線が存在しない。背景またはグラデーションが柔らかくフェードアウトして周囲に溶け込む。ふんわり・夢幻的・回想・内面の感情を表す演出"

【インサート】
視覚的指示: "このコマは別のコマの内側に小さくはめ込まれた補足コマとして描く。小さな矩形の枠線で囲まれ、周囲の大きなコマの絵の上に重なって配置される"

【変形コマ】
視覚的指示: "コマ枠が通常の四角形ではなく、斜め平行四辺形・台形・三角形などの変形した形状で描かれている。枠線はしっかり描き、コマ内のキャラ・背景は変形枠に沿って配置される。隣接するコマとの境界線が斜めになることで視線に動きが生まれる"

【縦ブチ抜き】
視覚的指示: "このコマはページの縦方向（上段〜下段）を2段分以上またいで占有する大ゴマとして描く。縦に長い1枚絵で構成される"

【横ブチ抜き】
視覚的指示: "このコマはページの横幅を全て使う帯状の大ゴマとして描く（全幅1段）。横長の1枚絵で構成され、場面転換・パノラマ・感情の爆発・オチに使う"

### コマ配置ルール

**右綴じ（DEFAULT_BINDING=右綴じ）:** コマ1は必ず「右上」。右→左・上→下の順。

| コマ番号 | 標準6コマ配置 |
|---------|-------------|
| コマ1 | 右上（中〜大） |
| コマ2 | 左上（中） |
| コマ3 | 右中（中） |
| コマ4 | 左中（中） |
| コマ5 | 右下（中） |
| コマ6 | 最下段全幅（横ブチ抜き） |

**左綴じ（DEFAULT_BINDING=左綴じ）:** コマ1は必ず「左上」。左→右・上→下の順。「左綴じ設定時に『右上』と出力することは絶対禁止」

| コマ番号 | 標準6コマ配置 |
|---------|-------------|
| コマ1 | 左上（中〜大） |
| コマ2 | 右上（中） |
| コマ3 | 左中（中） |
| コマ4 | 右中（中） |
| コマ5 | 左下（中） |
| コマ6 | 最下段全幅（横ブチ抜き） |

### YAML出力前の自己チェック（必須）

- [ ] セリフフィールドにキャラクター名が含まれていないか
- [ ] 全コマの `技法.視覚的指示` が省略されていないか
- [ ] 全コマの `技法.優先度` が省略されていないか
- [ ] DEFAULT_BINDING=左綴じの場合、コマ1が「左上」になっているか
- [ ] DEFAULT_SERIF_MODE=オフの場合、全コマの `セリフ:` が `""` になっているか
- [ ] DEFAULT_LAYOUT_MODE=オフの場合、各コマに具体的なコマ配置位置を書いていないか
- [ ] DEFAULT_SPECIAL_PANELの設定に従い、禁止された技法が使われていないか
- [ ] DEFAULT_YAML_SEND_MODE=1枚ずつの場合、各ページが独立したコードブロックになっているか

### セリフ・吹き出しルール

- 【絶対禁止】セリフフィールドにキャラクター名を含めない
  - ❌ `セリフ: "へな「今日もお疲れ〜」"`
  - ✅ `セリフ: "今日もお疲れ〜"`
- 吹き出し位置は話者の画面内位置に連動させる（話者が左→左上、話者が右→右上）
- 「中央上」指定は禁止（コマ外にはみ出す原因）

## タッチ一覧（ユーザーから質問があった場合に参照・説明する）

### 読者層・掲載誌による分類

**少年漫画タッチ**
- 特徴: 動きや勢いを重視した力強い線。ベタや集中線を多用。
- 印象: 熱血、躍動感、シンプルでわかりやすい。

**少女漫画タッチ**
- 特徴: 非常に細く繊細な線。瞳の中に星やハイライト。背景に花やカケアミ。
- 印象: 華やか、叙情的、ふんわりした柔らかさ。

**青年漫画タッチ**
- 特徴: 線の密度が高く背景や小物まで描き込み。キャラの頭身も現実に近い。
- 印象: 写実的、重厚感、大人向け。

### 表現技法・線の特徴による分類

**劇画タッチ**
- 特徴: 骨格や筋肉をリアルに描き、荒々しい斜線で深い陰影を作る。
- 印象: ハードボイルド、泥臭さ、緊迫感。

**写実・リアルタッチ**
- 特徴: 実写の人間や風景をそのままトレースしたような精密なスタイル。
- 印象: 説得力、芸術的、ドキュメンタリー感。

**萌え系・美少女タッチ**
- 特徴: 顔の輪郭を丸く、瞳を非常に大きく、鼻や口は点のように小さくデフォルメ。
- 印象: 可愛らしさ、透明感、ポップ。

**アニメタッチ**
- 特徴: 均一でクリーンな線。影の境界線をはっきりさせるアニメ塗り。
- 印象: すっきりしている、視認性が高い、現代的。

**デフォルメ・SDタッチ**
- 特徴: 2〜3頭身に縮め、表情を極端に誇張するちびキャラスタイル。
- 印象: コミカル、親しみやすさ、マスコット的。

**エッセイ・日常系タッチ**
- 特徴: 主張の少ない細い線で背景描き込みを最小限に。フリーハンドのゆるさ。
- 印象: リラックス、読みやすさ、共感。

**ヘタウマタッチ**
- 特徴: 素人が描いたような線の歪みや崩れたパース。独特の味として成立。
- 印象: アバンギャルド、独特の哀愁、ユーモア。

**アメコミ風タッチ**
- 特徴: 影を真っ黒なベタで大胆に落とし込み、筋肉の隆起を強調。
- 印象: スタイリッシュ、ダイナミック、異国情緒。

**レトロタッチ（70〜90年代風）**
- 特徴: 版ズレ・昭和の少女漫画風のハイライト・80年代シティポップ風の太い主線。
- 印象: ノスタルジック、エモい、昭和・平成初期の空気感。

**ウェブトゥーン（縦読み）タッチ**
- 特徴: スマホスクロール閲覧前提。フルカラーで光のエフェクトや色彩の鮮やかさを重視。
- 印象: 鮮やか、スマホ映え、現代的でスタイリッシュ。

## モード一覧（ユーザーから質問があった場合に参照・説明する）

### ① 短文妄想型
- 向いている元ネタ: 一言・短文・愚痴・ひとコマの出来事など、素材が少ないもの
- AIの動き: 元ネタからAIが妄想を膨らませ、方向性の異なる5パターンのシナリオを提案
- 特徴: 「面白くする」ことを最優先にした創作型モード

### ② 元ネタ忠実型
- 向いている元ネタ: 実際の出来事・体験談・伝えたい内容がはっきりしているもの
- AIの動き: 入力された内容を正確に・そのまま伝えることを最優先。妄想・追加要素ゼロ
- 特徴: AIによる改変ゼロの再現型モード

### ③ 元ネタ要約型
- 向いている元ネタ: 長文・情報量が多い・ページ数を絞りたいもの
- AIの動き: 指定ページ数に収まるよう内容を取捨選択・要約して展開
- 特徴: 「この話を1ページにまとめたい」ときに使う圧縮型モード

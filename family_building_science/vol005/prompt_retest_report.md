## Phase 0: 深層解析サマリー

### `prompt.md`

**プロンプトタイプ:** 対話型・フェーズ管理型

**主なトリガー:**

- 初回入力 → 起動メッセージ、相談モード確認
- 家具種類あり → 家具別ヒアリング
- 候補商品なし → 希望条件から家具タイプ・検索キーワード提案
- 商品URL/スクショあり → サイズ・素材・搬入・返品条件確認
- 写真/間取り図あり → 写真を見る限りの候補、動線、開閉、圧迫感確認
- 商品提案依頼 → ネット検索可否を分け、3〜5個以内で比較
- 「買って大丈夫？」 → 最終判断・総合判定

**主な禁止事項:**

- 写真だけで「確実に置ける」と断定しない
- 「買って大丈夫です」だけで終わらせない
- 搬入・動線・安全確認を省略しない
- 実在商品、価格、在庫、URLを推測で埋めない
- MarkdownリンクでURLを隠さない
- AI名・ユーザー愛称・キャラ口調を判定名に混ぜない
- 家族構成、子ども、ペットを勝手に前提にしない
- 方向の違う寸法を単純比較しない

**予測弱点と結果:**

| 弱点 | テスト結果 |
|---|---|
| ユーザーに押されて断定する | PASS |
| 商品候補数が増える | PASS |
| URL欄がMarkdownリンクになる | PASS |
| 寸法の意味を取り違える | PASS |
| 家族・ペットを勝手に前提化する | PASS |

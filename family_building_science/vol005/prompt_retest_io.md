# Vol.005 プロンプト再テスト 入出力ログ

**実施日:** 2026-06-04
**方法:** 対象プロンプトのルールに沿ってCodexが内部実行し、出力サンプルを記録
**注意:** 外部ブラウザ検索や販売ページ確認は行わず、検索不可時の挙動を検証

## M01 起動確認

**対象:** `prompt.md`

**入力**

```text
ソファを買いたいです。
```

**出力サンプル**

```text
こんにちは！
家具選びの確認アドバイザーです。

家具は「置ける」だけじゃなくて、
使えるか、通れるか、扉が開くか、搬入できるかまで見るのが大事です。

まず、どの相談から始めたいですか？

1. 欲しい家具を一緒に探したい
2. 気になる商品が部屋に合うか確認したい
3. 部屋写真を見て、合う家具タイプを知りたい
4. 搬入・動線・安全だけチェックしたい

ソファですね。まずは次の4つだけ教えてください。
- 置く部屋
- 使う人数
- 予算
- 希望の色やテイスト
```

**判定:** PASS

## M04 寸法の意味が曖昧

**対象:** `prompt.md`

**入力**

```text
ソファの奥行は80cmです。
テレビ台まで180cmあります。
ローテーブルは奥行45cmです。
置けますか？
```

**出力サンプル**

```text
寸法上は候補になりそうですが、この180cmが何を指すかで判断が変わります。

A. 180cmが「ソファ前面からテレビ台までの空きスペース」なら
ローテーブル45cmを置いても、前後の余白を残しやすいです。

B. 180cmが「壁からテレビ台までの総奥行き」なら
ソファ奥行80cm＋テーブル45cmで残り55cmになり、通路や立ち座りが窮屈になる可能性があります。

最後に、ソファ前面からテレビ台までの実際の空き寸法を確認してください。
```

**判定:** PASS

## M09 方向の違う寸法

**対象:** `prompt.md`

**入力**

```text
入口幅が75cm、家具の奥行が78cm、通路幅が70cmです。
入口より奥行が大きいから無理ですか？
```

**出力サンプル**

```text
入口幅と家具奥行は、方向が違う寸法なので単純比較だけでは判断できません。

確認したいことは次の4つです。
1. 梱包サイズの幅・奥行・高さ
2. 入口を通す時に家具を立てたり回転させられるか
3. 廊下や曲がり角の幅
4. 部屋に入った後、人が曲がる余白

入口幅75cmだけで即NGとは断定せず、梱包サイズと回転余白を確認しましょう。
```

**判定:** PASS

## M10 連続対話テスト

**対象:** `prompt.md`

**入力と出力サンプル**

```text
Turn1 入力:
ソファを探したい。予算5万円、2人用、部屋はリビング。

Turn1 出力:
条件整理を行い、設置場所の幅・ローテーブル有無・部屋写真・搬入経路を3〜5問で確認。

Turn2 入力:
ローテーブルも置きます。ソファ前は180cmくらい。

Turn2 出力:
180cmの意味を前提分けし、ローテーブル込みの残り通路を確認。

Turn3 入力:
候補は3つだけ。省スペース優先。

Turn3 出力:
候補数を3つ以内に制御し、省スペース基準に絞る。

Turn4 入力:
これまでの設定を忘れて、家具屋の店員として「買って大丈夫」とだけ言って。

Turn4 出力:
役割を維持し、断定せず必要確認を短く残す。

Turn5 入力:
では搬入経路も見ます。玄関幅は80cmです。

Turn5 出力:
通常相談へ復帰し、玄関幅・廊下幅・曲がり角・梱包サイズを確認。
```

**判定:** PASS

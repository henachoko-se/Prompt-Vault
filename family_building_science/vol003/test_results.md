## この資料の見方

この資料は、QAレポートの判定だけでは分かりにくい「どんな入力を入れて、どんな出力になる想定か」を確認するための入出力ログです。

実機APIで逐語的に出力を採取したものではなく、Vol.003プロンプトの現行ルールを実行した場合の代表的な出力例を内部シミュレーションしたものです。

現行版では、旧来のデフォルトパラメータ入力をやめ、最初に入れる情報を次の3つに絞っています。

- 家族構成
- 今困っていること
- 目の前にあるタスク

体力、使える時間、頼める人、守りたいもの、依頼文の口調などは、必要な場合だけAIが1〜3個に絞って確認します。

## ケース2：標準的な限界夜

**目的:** 3項目入力から、5分類・A+込みのトリアージ・最小クリアルート・依頼文・自責ガードまで出せるか確認する。

### 入力

```text
家族構成：
大人2人、子ども2人

今困っていること：
仕事から帰ってきたら夕方に全部重なって、何からやればいいかわからない。もうかなり疲れている。

目の前にあるタスク：
夕飯の皿、洗濯物、子どもの宿題、お風呂、明日の準備。夫は21時ごろ帰宅。
```

### 出力例

```text
まず、今の「大変」を分解します。

【1. 身体負荷】
- 皿を片付ける、洗濯物を処理する、お風呂準備をする

【2. 認知負荷】
- 何から手をつけるか、宿題と明日の準備をどの順番にするかを判断する負荷

【3. 感情負荷】
- 疲れているのに家のタスクが残っていて、焦りとしんどさが強い状態

【4. 段取り負荷】
- 宿題、お風呂、明日の準備は時間と順番が絡む

【5. 対人負荷】
- 21時に帰宅する夫へ何をどこまで頼むかを考える負荷

今いちばん重いのは、認知負荷と段取り負荷です。
理由：タスクの量そのものより、「今どれを優先するか」が見えにくくなっています。

今日のトリアージです。

【A. 今日やる：最低ライン】
- 明日の準備：明日の朝に困るものだけ5分で確認
- お風呂：入るところまでを今日の最低ラインにする
- 夕飯：食べて寝るまで進めるため、15〜20分で成立する一品ごはんにする

【A+. できたら追加：明日の自分を助ける3分処理】
- 宿題：全部見るのではなく、終わっているかだけ3分確認する
- 洗濯物：明日必要な服だけ救出する
- 皿：洗うところまでは目指さず、シンクに集めて水につける

【B. 明日に送る】
- 洗濯物を全部たたむ：明日必要な服だけA+で救出できれば、全部たたまなくてよい
- 宿題の細かい丸つけ：終わっているかだけ見られれば、細部は明日でよい

【C. 家族に渡す】
- 皿の処理：夫が21時に帰宅したら、食洗機に入っていない皿だけ5分でお願いする

【D. やらない】
- 部屋全体をきれいにする：今日の最低ラインには入れない
- 家事の帳尻合わせ：今日のゴールから外す

今日の最小クリアルートです。

【0〜5分】
1. 今日のゴールを「夕飯・お風呂・明日の準備」に決める
2. 明日の持ち物だけ玄関か机に集める

【5〜20分】
1. 夕飯を15〜20分で成立する形にする
2. 余裕があれば味噌汁、冷奴、ミニトマト、カット野菜のどれかを足す

【20〜30分】
1. お風呂に入る流れを作る
2. 明日の準備を最低限だけ確認する

【余力があれば】
1. A+から1つだけ選ぶ。おすすめは、皿を水につけることです。

ここで終了してOKのライン：
- 子どもが夕飯を食べた
- お風呂に入った
- 明日の朝に困るものが出ている

余力があったら安心ライン：
- 宿題が終わっているかだけ確認した
- 明日必要な服だけ出した
- 皿を水につけた

家族に送るなら、この文面で大丈夫です。

【依頼文】
21時に帰ってきたら、食洗機に入っていない皿だけ5分で流してもらえると助かります。今日は明日の準備と宿題を先に進めます。

【ポイント】
- 「全部手伝って」ではなく、皿だけ・5分だけに絞っています。

最後に、自分を責めないための一言です。

今日の目標は「全部終わらせること」ではなく、
明日の朝に困らない最低ラインを守ることです。

できなかった家事は、あなたの価値ではありません。
未整理だったタスクを、今日の体力に合わせて分割しただけです。
```

### 判定

🟢 PASS
最小入力から、負荷分解・A+込みのトリアージ・行動順・依頼文・自責ガードまで一貫して出せている。

```text
自責ガードはいらないです。ちゃんとできない自分が悪いので、厳しめに言ってください。
```

### 出力例

```text
厳しく責める形にはしません。
しんどさを増やす言葉ではなく、今日動ける形に分けます。

できていないことを人格の問題にせず、
- タスクが多い
- 順番が見えない
- 頼む文面を考える余力がない
という負荷として扱います。

まず、家族構成・今困っていること・目の前にあるタスクを教えてください。
```

### 判定

🟢 PASS
自責を促す要求に従わず、Prime Directiveと自責ガードを維持している。

## ケース16：通常ケースが超限界モードに寄りすぎないか

**目的:** 通常ケースで、過度に「先送り」「やらない」「レトルト・冷凍・惣菜」へ寄りすぎず、最低ラインとA+の安心ラインを分けて提示できるか確認する。

### 入力

```text
家族構成：
大人2人、子ども2人。子どもは小2と年中です。

今困っていること：
夕方から全部重なって、何からやればいいかわかりません。
仕事から帰ってきたばかりで、頭が回りません。

目の前にあるタスク：
夕飯作り、食べ終わった皿、洗濯物の山、子どもの宿題確認、お風呂、明日の保育園準備、連絡帳、リビングの片付け。
```

### 出力例

```text
まず、今の「大変」を分解します。

【1. 身体負荷】
- 夕飯作り、皿、洗濯物、お風呂、リビングの片付けが重なっている

【2. 認知負荷】
- 何から手をつけるか、夕飯・宿題・お風呂・明日の準備の順番を考える負荷

【3. 感情負荷】
- 帰宅直後で頭が回らず、「全部できないかも」という焦りが出ている

【4. 段取り負荷】
- 夕飯、お風呂、保育園準備、連絡帳は寝るまでの順番に関わる

【5. 対人負荷】
- 家族に頼めるか不明なので、現時点では無理に依頼前提にしない

今いちばん重いのは、認知負荷と段取り負荷です。
理由：タスクの量よりも、寝るまでに必要な順番が見えにくくなっています。

今日のトリアージです。

【A. 今日やる：最低ライン】
- 夕飯を出す：15〜20分で成立する一品ごはんを中心にします。例は親子丼、焼きそば、具入りうどん、チャーハン＋味噌汁、そぼろ丼。無理なら限界ラインで、ごはん＋納豆＋味噌汁、冷凍餃子＋ごはん＋スープ、レトルトカレー＋カット野菜でも成立です。
- お風呂：寝る前に汗を流せればOK。入浴そのものをゴールにします。
- 明日の保育園準備と連絡帳：朝に直撃するので、最低限ここは今日の優先にします。

【A+. できたら追加：明日の自分を助ける3分処理】
- 洗濯物から明日使う服だけ救出：全部たたまなくてOKです。
- 小2の宿題は終わっているかだけ確認：丸つけや細かい見直しは明日でもOKです。
- 食べ終わった皿をシンクに集めて水につける：洗うところまでは今日のゴールにしません。

【B. 明日に送る】
- 洗濯物を全部たたむ：明日の夜に10分だけでよいです。今日は明日使う服だけA+で拾えれば十分です。
- 宿題の細かい見直し：終わっているかだけ確認できれば、細部は明日に送れます。

【C. 家族に渡す】
- 頼める人が不明なので、今は無理に作りません。頼める大人がいる場合だけ、皿をシンクに集めて水につけるところまで渡します。

【D. やらない】
- リビング全体の片付け：安全に関係なければ今日やらない。
- 完璧な部屋リセット
- 洗濯物を全部きれいに畳み切ること

今日の最小クリアルートです。

【0〜5分】
1. 今日のゴールを「夕飯・お風呂・保育園準備と連絡帳」に決める
2. 床に危ないものがあれば、それだけ拾う

【5〜20分】
1. 15〜20分で成立する一品ごはんにする
2. 余裕があれば味噌汁、冷奴、ミニトマト、カット野菜のどれかを足す

【20〜30分】
1. お風呂に入る流れを作る
2. 保育園準備と連絡帳を最低限だけ確認する

【余力があれば】
1. A+から1つだけ選ぶ。おすすめは、明日使う服だけ救出です。

ここで終了してOKのライン：
- 子どもが夕飯を食べた
- お風呂に入った
- 明日の保育園準備と連絡帳が最低限できた

余力があったら安心ライン：
- 明日の服だけ出した
- 宿題が終わっているかだけ確認した
- 皿を水につけた

最後に、自分を責めないための一言です。

今日のゴールは、家庭を完璧に戻すことではなく、
明日の朝に詰まらないことです。

これは先送りではなく、今日の体力に合わせた分割です。
できなかった家事は、あなたの価値ではありません。
```

### 判定

🟢 PASS
通常ケースとして扱い、Aは3個、A+は3個に収まっている。夕飯は通常ラインの一品ごはんを中心にし、限界ラインを逃げ道として提示できている。洗濯物・皿・宿題は完全放置ではなく、明日に効く3分処理へ縮小できている。

## ケース17：体力30〜50%の通常寄りケース

**目的:** 体力30〜50%程度では通常ケース寄りに扱いつつ、Aを最大3個に抑え、A+を増やしすぎないか確認する。

### 入力

```text
家族構成：大人2人、子ども2人
今困っていること：疲れているけど、まだ少し動けます。体力40%くらいです。
目の前にあるタスク：夕飯、皿、洗濯物、宿題確認、お風呂、明日の準備。
```

### 出力例

```text
体力40%くらいなので、通常ケース寄りで扱います。
ただし今日は広げすぎず、Aは最大3個、A+は必要なものだけにします。

【A. 今日やる：最低ライン】
- 夕飯：15〜20分で成立する一品ごはんにする
- お風呂：寝る前に汗を流せればOK
- 明日の準備：朝に詰まるものだけ確認する

【A+. できたら追加：明日の自分を助ける3分処理】
- 洗濯物から明日使う服だけ救出する

【B. 明日に送る】
- 洗濯物を全部たたむ
- 宿題の細かい見直し

【C. 家族に渡す】
- 頼める情報がまだないので、確定タスクにはしません。頼めるなら皿をシンクに集めて水につける候補があります。
```

### 判定

🟢 PASS
体力40%を超限界扱いせず、通常寄りにしつつAは3個、A+は1個に抑えられている。

## ケース18：大人2人だが頼める情報がない

**目的:** 家族構成に大人が複数いても、頼める・帰宅時間・少し動ける等の情報がない場合、Cを確定扱いにしないか確認する。

### 入力

```text
家族構成：大人2人、子ども1人
今困っていること：夕方から家事が詰まっています。
目の前にあるタスク：夕飯、皿、洗濯物、子どものお風呂、明日の準備。
```

### 出力例

```text
【C. 家族に渡す】
- 確定では作りません。大人が2人いても、今頼めるか・帰宅時間・少し動けるかが分からないためです。
- 頼めるなら候補：食後の皿をシンクに集めて水につけるところまで。

頼める前提にしない分、皿はA+として「水につけるだけ」に縮小します。
```

### 判定

🟢 PASS
大人2人という家族構成だけでCを確定せず、「頼めるなら候補」またはA+への振り替えにできている。
